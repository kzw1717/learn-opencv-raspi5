# 用語解説：ファイルオブジェクト

> `leaf_monitor.py` でログを書くとき、`with open(LOG_CSV, "a") as f:` の `f` が出てきます。
> この `f` が**ファイルオブジェクト**——「開いたファイルそのものを表す“もの”」です。読む・書くといった操作は、この `f` に対して行います。

---

## 1. これは何？（考え方）

`open(...)` を実行すると、ファイルが開かれ、その「開いたファイル」を表すオブジェクトが返ってきます。これが**ファイルオブジェクト**です。

> ファイルそのものではなく、「いま開いているファイルへの**取っ手（ハンドル）**」だと思ってください。

この取っ手（`f`）を通して、「読む」「書く」「1行ずつたどる」といった操作をします。ファイルオブジェクトもオブジェクトの一種なので、`.read()`・`.write()` などのメソッドを持っています（→「オブジェクト」の解説も参照）。

---

## 2. 基本の形

```python
# 開く → ファイルオブジェクト f が返る
with open("ファイル名", "モード") as f:
    f.write("文字列")      # 書き込む
    # または
    text = f.read()        # 全部読む
    for line in f:         # 1行ずつたどる
        ...
```

おもなメソッドとモード：

| メソッド | 役割 |
|---|---|
| `f.write(文字列)` | 文字列を書き込む |
| `f.read()` | ファイル全体を読み込む |
| `f.readline()` / `for line in f` | 1行ずつ読む |

| モード | 意味 |
|---|---|
| `"r"` | 読み込み（既定） |
| `"w"` | 上書き（中身を消して書く） |
| `"a"` | 追記（末尾に付け足す） |

`with` と一緒に使うと、閉じ忘れの心配がありません（→「with」の解説を参照）。

---

## 3. サンプルスクリプト

`leaf_monitor.py` のログ（`log.csv`）のように、「1行ずつ追記して、あとで読み返す」流れを小さく再現してみましょう。

```python
# sample_fileobj.py

# 追記モードで開く → ファイルオブジェクト f に書く
with open("log_demo.csv", "a") as f:
    f.write("2025-06-19_08:00, 5.2\n")
    f.write("2025-06-19_12:00, 7.8\n")
# ここで自動的に閉じられる

# 読み込みモードで開く → 1行ずつたどる
with open("log_demo.csv") as f:
    for line in f:                 # f を for で回すと1行ずつ取れる
        print("行:", line.strip()) # strip() で末尾の改行を消す
```

実行結果（例）：

```
行: 2025-06-19_08:00, 5.2
行: 2025-06-19_12:00, 7.8
```

`for line in f:` で、ファイルオブジェクトを1行ずつたどれる点に注目してください。`"a"`（追記）なので、何度も実行すると行が増えていきます。

> 改行コード `\n` を自分で付けているのは、`write` が自動では改行しないからです。

---

## 4. 代表的な使い方

- **ログを追記する**：`"a"` で開いて `f.write(...)`。計測のたびに1行ずつ積み上げる。
- **設定や結果を読み込む**：`"r"`（既定）で開いて `f.read()` や `for line in f`。
- **CSV・JSON ライブラリに渡す**：`csv.writer(f)` や `json.dump(..., f)` のように、ファイルオブジェクト `f` を専用ライブラリに渡して扱う。

> **やってみよう**
> `sample_fileobj.py` を2〜3回実行して、`log_demo.csv` の行が増えていくのを確認しましょう。次に `"a"` を `"w"` に変えて実行すると、毎回1行に上書きされます。追記と上書きの違いを体感してください。

---

## 5. leaf_monitor.py での実例

**(1) CSV ライブラリにファイルオブジェクトを渡す** — `append_log()`

```python
with open(LOG_CSV, "a", newline="") as f:
    w = csv.writer(f)          # f（ファイルオブジェクト）を csv.writer に渡す
    if new_file:
        w.writerow(["timestamp", "coverage_pct", "clumps"])
    w.writerow([timestamp, f"{coverage:.3f}", clumps])
```

`open(...)` で得たファイルオブジェクト `f` を、CSV 用の道具 `csv.writer` に渡しています。直接 `f.write` するより、CSV の体裁を整えてくれて便利です。`newline=""` は、CSV で余計な空行が入るのを防ぐためのお作法です。

**(2) JSON ライブラリにファイルオブジェクトを渡す** — `calibrate()` / `load_mat_config()`

```python
with open(MAT_CONFIG, "w") as f:
    json.dump({...}, f, indent=2)   # f に JSON を書き出す

with open(MAT_CONFIG) as f:
    cfg = json.load(f)              # f から JSON を読み込む
```

ここでも、ファイルオブジェクト `f` を JSON 用の道具に渡しています。「ファイルオブジェクトを開いて、専用ライブラリに渡す」のは、とてもよくあるパターンです。

---

## 6. まとめ

- ファイルオブジェクトは「開いたファイルへの取っ手」。`open(...)` が返す。
- `f.write(...)` で書き、`f.read()` や `for line in f` で読む。
- モードは `"r"`（読込）・`"w"`（上書き）・`"a"`（追記）。
- `csv.writer(f)` や `json.dump(..., f)` のように、専用ライブラリに渡して使うことも多い。
- `with` と組み合わせると閉じ忘れがなく安全。
- `leaf_monitor.py` は、ログ（CSV）と設定（JSON）の読み書きでファイルオブジェクトを使っている。
