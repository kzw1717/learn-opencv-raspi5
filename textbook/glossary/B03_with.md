# 用語解説：with（コンテキストマネージャ）

> `leaf_monitor.py` でファイルを読み書きするところは、すべて `with open(...) as f:` という形になっています。
> この `with` は「開いたら必ず閉じる」を自動でやってくれる、安全のための仕組みです。

---

## 1. これは何？（考え方）

ファイルは、「開く → 使う → **閉じる**」の3ステップで扱います。問題は、最後の「閉じる」を忘れがちなことです。閉じ忘れると、書いた内容がきちんと保存されなかったり、ファイルが開きっぱなしで他の処理の邪魔になったりします。

`with` は、この面倒で忘れやすい「閉じる」を**自動でやってくれる**書き方です。

> `with` のブロック（インデントされた範囲）を抜けると、ファイルは自動で閉じられる。

途中でエラーが起きても、ちゃんと閉じてくれます。だから現代の Python では、ファイルを扱うときは `with` を使うのが基本です。

---

## 2. 基本の形

```python
with open("ファイル名", "モード") as 変数:
    # この中でファイルを使う
    ...
# ここに来た時点で、ファイルは自動で閉じられている
```

`open` のおもな「モード」：

| モード | 意味 |
|---|---|
| `"r"` | 読み込み（read）。省略時の既定。 |
| `"w"` | 書き込み（write）。**中身を消して**新しく書く。 |
| `"a"` | 追記（append）。末尾に**付け足す**。 |

`leaf_monitor.py` のログ追記では `"a"`（追記）が使われています。

---

## 3. サンプルスクリプト

`with` を使うと、何がうれしいのかを比べてみましょう。

```python
# sample_with.py

# ── with を使わない書き方（閉じ忘れの危険がある）──
f = open("memo1.txt", "w")
f.write("マットは青\n")
f.close()        # ← これを忘れると保存されないことがある

# ── with を使う書き方（自動で閉じる。こちらが推奨）──
with open("memo2.txt", "w") as f:
    f.write("落ち葉モニタリング\n")
    f.write("被覆面積率を記録\n")
# ここで自動的に閉じられている（f.close() は不要）

# 読み込みも with で
with open("memo2.txt") as f:
    print(f.read())
```

実行すると `memo2.txt` の中身が表示されます。`with` のほうは `close()` を書いていないのに、ちゃんと保存されている点に注目してください。

---

## 4. 代表的な使い方

- **ファイルの読み書き**：いちばん典型的な使い方。`with open(...) as f:`。
- **CSV や JSON の保存・読込**：`csv.writer(f)` や `json.dump(..., f)` と組み合わせる。
- 一般に「使う前に準備し、使い終わったら後始末が必要なもの」を安全に扱える仕組み（ファイル以外にもある）。

> **やってみよう**
> `sample_with.py` の `"w"` を `"a"`（追記）に変えて2回実行し、`memo2.txt` の行が増えていくことを確認しましょう。`"w"` だと毎回上書き、`"a"` だと付け足し、という違いが分かります。

---

## 5. leaf_monitor.py での実例

**(1) ログを追記する** — `append_log()`

```python
with open(LOG_CSV, "a", newline="") as f:
    w = csv.writer(f)
    if new_file:
        w.writerow(["timestamp", "coverage_pct", "clumps"])
    w.writerow([timestamp, f"{coverage:.3f}", clumps])
```

`"a"`（追記）モードで開き、計測結果を1行ずつ末尾に付け足しています。ブロックを抜ければ自動で閉じるので、`close()` は書かれていません。計測のたびに安全にログが積み上がっていきます。

**(2) マット色設定を保存・読込** — `calibrate()` と `load_mat_config()`

```python
with open(MAT_CONFIG, "w") as f:
    json.dump({...}, f, indent=2)

with open(MAT_CONFIG) as f:
    cfg = json.load(f)
```

JSON の保存（`"w"`）と読み込み（既定の `"r"`）の両方で `with` が使われています。

---

## 6. まとめ

- `with` は「開いたら必ず閉じる」を自動化する書き方。閉じ忘れを防ぐ。
- 形は `with open("ファイル", "モード") as 変数:`。ブロックを抜けると自動で閉じる。
- モードは `"r"`（読込）・`"w"`（上書き）・`"a"`（追記）が基本。
- `leaf_monitor.py` はログ追記・設定保存・設定読込のすべてで `with` を使っている。
