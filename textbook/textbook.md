## 0.1 私たちがたどり着く場所

`leaf_monitor.py` は、Raspberry Pi のカメラで鉢の周りを定期的に撮影し、**色付きのマットの上に積もった落ち葉やゴミ（堆積物）が、マットの何パーセントを覆っているか**を自動で計算して記録するプログラムです。この「覆っている割合」を時間とともに記録していくことで、ミニトマトの落葉の傾向＝健康状態のトレンドを見守ることができます。

- **緑の線**で「見つけた堆積物（落ち葉など）」が囲まれている
- 左上に `coverage=12.34%  clumps=5` のように、覆われた割合と塊の数が書かれている

私たちの目標は、この1枚の画像が**どうやって作られているのか**を、最初から最後まで理解することです。難しそうに見えますが、分解すれば一つひとつは素朴な作業の積み重ねです。

## 0.6 NumPy の最低限のおさらい

OpenCV の画像は、その正体が **NumPy の配列**です。WSL2 で少し触ったことがあるはずですが、このテキストで出てくる範囲だけ、軽くおさらいしておきます。詳しくは各章で必要になったときに改めて説明します。

| 書き方 | 意味 |
|---|---|
| `arr.shape` | 配列の形（大きさ）。画像なら `(縦, 横, 色数)` |
| `arr.dtype` | 1マスに入る数字の種類。画像は `uint8`（0〜255の整数） |
| `arr[100, 200]` | 上から100、左から200の位置の値を取り出す |
| `arr[:, :, 0]` | 3つ組のうち0番目だけを取り出す（スライス） |
| `np.zeros(形, dtype=np.uint8)` | すべて0（真っ黒）の配列を作る |
| `np.array([...])` | リストから配列を作る |

これくらいが分かっていれば、各章を読み進められます。「画像＝NumPy 配列」という感覚だけ、強く持っておいてください。

## 1.4 画像を読み込んで「形」を見る

では Python を書きます。`step1.py` というファイルを作りましょう。

```python
# step1.py
import cv2

# 画像をファイルから読み込む（中身は数字の表になって返ってくる）
img = cv2.imread("sample.jpg")

print(type(img))   # 何という種類のデータか
print(img.shape)   # 表の「形」＝大きさ
print(img.dtype)   # 1マスに入っている数字の種類
```

ターミナルで実行します。

```bash
python3 step1.py
```

出力はこんな感じになります（数字は撮った写真によって変わります）。

```
<class 'numpy.ndarray'>
(1520, 2028, 3)
uint8
```

1行ずつ読み解きます。

- `numpy.ndarray`：読み込んだ画像の正体は **NumPy の配列**、つまり数字の表です。普段 WSL2 で NumPy を触っているなら、まさにあれです。「画像が来た」のではなく「数字の表が来た」と考えてください。
- `(1520, 2028, 3)`：表の形です。**縦 1520 行 × 横 2028 列**、そして各マスに **3 個** の数字が入っている、という意味です。順番が「縦, 横」であって「横, 縦」でないことに注意してください。
- `uint8`：1つ1つの数字は **0〜255 の整数**です（`u`＝符号なし、`int8`＝8ビット整数）。0 が一番暗く、255 が一番明るい、という明るさの目盛りです。

3 という数字が出てきたのは、カラー画像が「赤・緑・青」の3つの明るさの組み合わせでできているからです。次の節でここを詳しく見ます。

## 1.6 初心者が必ずハマる罠：OpenCV は BGR

さて、ここがこの章で一番大事な注意点です。

さきほどの `[52, 138, 90]` の3つの数字。普通に考えると「赤・緑・青（RGB）の順かな?」と思いますよね。**ところが OpenCV は順番が逆で、青・緑・赤（BGR）の順**で数字を持っています。

- `pixel[0]` → **B**lue（青）の明るさ
- `pixel[1]` → **G**reen（緑）の明るさ
- `pixel[2]` → **R**ed（赤）の明るさ

これを知らずに「0番目は赤」と思い込むと、色の判定がまるごと狂います。実際に、間違って RGB のつもりで色を入れ替えるとどうなるか、目で確かめてみましょう。

```python
# step3.py
import cv2

img = cv2.imread("sample.jpg")

# わざと青(B)と赤(R)を入れ替えてみる
swapped = img[:, :, ::-1]   # 3つ組の並びを逆さまにする

cv2.imwrite("swapped.jpg", swapped)
print("swapped.jpg を保存しました。VS Code で開いて比べてみてください。")
```

`sample.jpg` と `swapped.jpg` を VS Code で並べて開いてみてください。空が赤くなったり、肌が青くなったりして、**色が壊れている**はずです。これが「順番を間違えるとどうなるか」の正体です。

実はこの BGR の罠は、`leaf_monitor.py` の作者も強く意識していて、わざわざ「カメラの画像はいったん JPEG ファイルに保存してから読み込む」という回り道をしています。これは picamera2（カメラ用ライブラリ）が返す色の順番がややこしく、直接渡すと色が入れ替わる事故が起きやすいためです。**色の順番は、画像処理では命綱**だと覚えておいてください。

> **やってみよう 3**
> `swapped.jpg` をもう一度 `step3.py` と同じやり方で並べ替えたら、元の色に戻るでしょうか? なぜそうなるか、理由も考えてみましょう。

## 1.7 「数字の表」をいちばん実感できる方法：グレースケール

カラーだと「3つ組」で少し複雑です。色を捨てて**明るさだけ**にすると、画像が本当にただの数字の表だと実感できます。

```python
# step4.py
import cv2

img = cv2.imread("sample.jpg")

# カラー(BGR) → 白黒(明るさだけ)に変換
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

print(gray.shape)        # 形が変わる
print(gray[100, 200])    # 1マスの中身も変わる

cv2.imwrite("gray.jpg", gray)
```

実行すると、形がこう変わります。

```
(1520, 2028)
185
```

カラーのときは `(1520, 2028, 3)` で、各マスに3個の数字がありました。白黒にすると `(1520, 2028)` になり、**各マスの数字が1個**になります。色がなくなり、純粋に「明るさ（0〜255）の表」になったわけです。`gray.jpg` を VS Code で開けば、白黒写真として確認できます。

ここで初めて `cv2.cvtColor`（色空間の変換）という関数が登場しました。これは**第2章の主役**です。第2章では同じ `cv2.cvtColor` を使って、白黒ではなく「HSV」という、色を扱うのにとても便利な形へ変換していきます。`leaf_monitor.py` がマットの色を見分けられるのは、この変換のおかげです。

> **やってみよう 4**
> `gray.jpg` の明るいところ・暗いところで、`gray[行, 列]` の数字がどう違うか、いくつかの場所で確かめてみましょう。「明るい＝大きい数字」が体感できるはずです。

**(1) 画像を読み込む（数字の表にする）** — `run_once()` の中

# 第2章 色を分けて考える（HSV）

> 第1章で、画像は「BGR（青・緑・赤）の数字の表」だと分かりました。
> でも実は、この BGR のままでは「マットの青色だけを取り出す」のがとても難しいのです。
> この章では、色を扱いやすくする魔法の変換 **HSV** を学びます。`leaf_monitor.py` がマットの色を見分けられる、その心臓部です。

---

## 2.1 この章のゴール

この章を終えると、次のことができるようになります。

- BGR のままだと色の指定が難しい理由を説明できる
- **HSV**（色相・彩度・明度）が、色を「色味」と「明るさ」に分けて扱える仕組みだと理解できる
- `cv2.cvtColor` で画像を HSV に変換し、H・S・V それぞれを画像として確認できる
- 「照明が変わっても色味（H）はあまり動かない」ことを、自分の目で確かめる

---

## 2.2 なぜ BGR だと困るのか

たとえば「青いマットの部分だけを取り出したい」とします。BGR で考えると、青は `B` が大きい色です。では「`B` が 150 以上なら青」と決めれば良さそうに見えます。

ところが現実はそう簡単ではありません。同じ青いマットでも、

- 日なたの明るい部分：`B` がとても大きい（たとえば 230）
- 日かげの暗い部分：`B` が小さい（たとえば 90）

このように、**同じ色なのに明るさで数字が大きく変わってしまう**のです。BGR では「色味」と「明るさ」がぐちゃっと混ざっているため、「明るくても暗くても青いものだけ」をうまく言い表せません。

自然光の下で1日中撮影する `leaf_monitor.py` では、これは致命的です。朝・昼・夕方で明るさが変わるたびに、マットの判定がブレてしまいます。

---

## 2.3 HSV という考え方

そこで登場するのが **HSV** です。HSV は色を、次の3つの要素に分けて表します。

- **H（Hue・色相）**：色味そのもの。「赤っぽい・青っぽい・緑っぽい」という色の種類。
- **S（Saturation・彩度）**：鮮やかさ。大きいほど鮮やか、小さいほど白っぽい・灰色っぽい。
- **V（Value・明度）**：明るさ。大きいほど明るい、小さいほど暗い。

ここが肝心です。HSV では、**明るさは V だけが担当**します。だから、

> 日なたでも日かげでも、変わるのは主に V（明るさ）だけ。色味の H はだいたい同じ。

となります。「明るさが変わっても、色味で物を見分ける」——これがまさに `leaf_monitor.py` の戦略です。

> **OpenCV ならではの注意：H の範囲は 0〜180**
> 色相は本来 0〜360 度の円で表しますが、OpenCV では 1マスに収まる 0〜255 の数字で表すため、半分にして **0〜180** にしています。「赤は 0 付近」「緑は 60 付近」「青は 120 付近」が目安です。一方、S と V は 0〜255 です。範囲が H だけ違う点に注意してください。

## 2.4 cvtColor で HSV に変換する

このあとの章でずっと使う、マット＋落ち葉のサンプル写真を撮っておきましょう。色付きマット（青か緑が推奨）の上に、落ち葉や小さなゴミをいくつか置いた状態で撮ります。

```bash

```

撮れたら `mat.jpg` を VS Code で開いて、マットと落ち葉が写っていることを確認してください。では変換してみます。

```python
# step5.py
import cv2

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

print("もとの画像の形:", img.shape)
print("HSVの画像の形 :", hsv.shape)

# HSVのある1点をのぞいてみる
print("ある画素のHSV:", hsv[100, 200])
```

```bash

```

出力はたとえばこうなります。

```

HSVの画像の形 : (1520, 2028, 3)

```

形は同じ `(縦, 横, 3)` のままですが、**中身の3つ組の意味が変わった**ことが大事です。BGR では「青・緑・赤の明るさ」でしたが、HSV では「色相・彩度・明度」になっています。`[110, 180, 200]` なら「色相110（青あたり）・彩度180（そこそこ鮮やか）・明度200（明るめ）」という意味です。

## 2.5 H・S・V を1枚ずつ見てみる

「色味と明るさが分かれている」と言われても、まだピンと来ないと思います。**H・S・V をそれぞれ別の白黒画像として保存して、目で見比べて**みましょう。第1章で学んだスライス（`[:, :, 番号]`）を使います。

```python
# step6.py
import cv2

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# 3つ組のそれぞれを取り出す
h = hsv[:, :, 0]   # 色相
s = hsv[:, :, 1]   # 彩度
v = hsv[:, :, 2]   # 明度

cv2.imwrite("hsv_h.jpg", h)
cv2.imwrite("hsv_s.jpg", s)
cv2.imwrite("hsv_v.jpg", v)
print("hsv_h.jpg / hsv_s.jpg / hsv_v.jpg を保存しました。")
```

3枚を VS Code で開いて見比べてください。読み取り方は次のとおりです。

- **hsv_v.jpg（明度）**：ふつうの白黒写真のように見えます。明るい場所が白く、影が黒くなります。「明るさの情報」がここに集まっている証拠です。
- **hsv_h.jpg（色相）**：明るさとは無関係に、色の種類で濃淡が決まります。マットの部分は、日なたでも日かげでも**同じくらいの濃さ**で写るはずです。これが「色味は明るさに左右されにくい」という HSV の長所です。
- **hsv_s.jpg（彩度）**：鮮やかな色付きマットは明るく、白っぽい・灰色っぽい部分は暗く写ります。

> **やってみよう 1**
> マットの上の「日なたの部分」と「影の部分」を1か所ずつ選び、`hsv` 画像でその位置の H と V を表示して比べてみましょう（`print(hsv[行, 列])`）。**V は大きく違うのに、H は近い値**になっていれば、HSV の狙いが体感できたことになります。

# 第3章 マスクを作る（色の抜き出し）

## 3.1 この章のゴール

この章を終えると、次のことができるようになります。

- **マスク**とは「白＝対象、黒＝対象外」の白黒画像（型紙）だと説明できる

## 3.2 マスクってなに？

**マスク**は、もとの画像と同じ大きさの白黒画像です。

- 白（255）＝「ここが対象だよ」という印
- 黒（0）＝「ここは対象じゃないよ」という印

工作で使う「型紙」を思い浮かべてください。型紙の穴が開いているところ（白）だけ絵の具が通り、それ以外（黒）は通さない——あの感覚です。画像処理では、このマスクを使って「どこを処理するか／しないか」を指定します。

`leaf_monitor.py` の中でも、`region.png`（計測範囲）や `pot.png`（鉢）など、いくつものマスクが活躍します。マスクは画像処理の主役級の道具なので、ここでしっかりつかんでおきましょう。

---

## 3.3 cv2.inRange で範囲内を白くする

`cv2.inRange` は、「指定した範囲に入っている画素を白、入っていない画素を黒」にした**マスクを作る**関数です。

```python
mask = cv2.inRange(hsv, 下限, 上限)
```

`下限`・`上限` は、HSV それぞれの「ここからここまで」を表す3つ組です。たとえば「H が 100〜140、S が 80〜255、V が 50〜255 の画素を白くする」なら、下限が `[100, 80, 50]`、上限が `[140, 255, 255]` です。

第2章で撮った `mat.jpg` を使って、青いマットを抜き出してみましょう。

```python
# step8.py
import cv2
import numpy as np

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# 青あたり（H=120付近）を抜き出す。値はマットの色に合わせて後で調整する
lower = np.array([100, 80, 50])
upper = np.array([140, 255, 255])

mask = cv2.inRange(hsv, lower, upper)

print("マスクの形:", mask.shape)
print("マスクに含まれる値:", np.unique(mask))   # 0と255だけのはず

cv2.imwrite("mat_mask.jpg", mask)
print("mat_mask.jpg を保存しました。")
```

```bash
python3 step8.py
```

出力の例です。

```
マスクの形: (1520, 2028)
マスクに含まれる値: [  0 255]
```

ここで2つ大事なことがあります。第一に、マスクの形は `(縦, 横)` で、**3つ組ではなく1マス1個の数字**です（白黒だから色は要りません）。第二に、中身は **0 と 255 しかありません**。まさに白と黒だけの型紙です。

`mat_mask.jpg` を VS Code で開いてみてください。マットの部分が**白く**、それ以外（鉢・落ち葉・背景）が**黒く**なっていれば成功です。

> もし白い部分が少なすぎる／多すぎる場合は、`lower` と `upper` の数字がマットの色に合っていません。次の節で調整します。

## 3.4 しきい値を変えて遊ぶ（いちばん大事な実験）

この章の本番はここです。**範囲を変えると抜き出され方がどう変わるか**を、手で動かして体に入れます。

まず、自分のマットの実際の HSV を調べましょう。`mat.jpg` をVS Codeで開き、マットの中央あたりがだいたい画像のどの位置か見当をつけて、その HSV を表示します。

```python
# step9.py
import cv2

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# マットの中央あたり（座標は自分の写真に合わせて変える）
print("マット中央の HSV:", hsv[760, 1000])
```

たとえば `[118 170 190]` と出たら、マットの色相 H は 118 あたりだと分かります。この値を中心に、`step8.py` の範囲を作り直してみましょう。「中心 ± 余裕（マージン）」で範囲を作るのがコツです。

```python
# step10.py
import cv2
import numpy as np

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

center_h = 118        # ← step9.py で調べた値に置き換える

# マージン（許容幅）。ここを変えて結果の変化を見るのが今回の実験
H_MARGIN = 12
S_MARGIN = 70
V_MARGIN = 80

s, v = 170, 190       # マット中央の S, V（step9で調べた値）
lower = np.array([center_h - H_MARGIN, max(0, s - S_MARGIN), max(0, v - V_MARGIN)])
upper = np.array([center_h + H_MARGIN, min(255, s + S_MARGIN), min(255, v + V_MARGIN)])

mask = cv2.inRange(hsv, lower, upper)
cv2.imwrite("mat_mask2.jpg", mask)
print("下限:", lower, " 上限:", upper)
print("mat_mask2.jpg を保存しました。")
```

> **やってみよう 1**
> `H_MARGIN` を `12 → 30` に増やして実行し、`mat_mask2.jpg` がどう変わるか見てください。白い部分が**広がる**はずです。範囲をゆるくすると、マットに近い色のものまで「マット」と判定されるようになります。

> **やってみよう 2**
> 逆に `H_MARGIN` を `12 → 4` に減らすと、白い部分が**狭く**なり、マットの一部が黒く欠けることもあります。範囲を厳しくしすぎると、本来のマットまで取りこぼします。

この「広げすぎると余計なものまで拾う／狭めすぎると取りこぼす」というトレードオフが、まさに `leaf_monitor.py` のチューニングの勘どころです。仕様書のトラブルシューティング表にある「誤検出が多ければマージンを広げ、取りこぼしが多ければ狭める」は、この実験そのものなのです。

> **やってみよう 3**
> `S_MARGIN`・`V_MARGIN` も変えてみましょう。S を厳しくすると、白っぽい・くすんだ部分が外れやすくなります。V を広げると、影になったマットも拾いやすくなります。`leaf_monitor.py` で V_MARGIN が広め（80）に設定されているのはなぜか、考えてみましょう。

---

## 3.5 赤い色のちょっとした特殊事情（軽く知っておく）

ここは「へぇ」と思う程度で大丈夫です。第2章で「H は 0〜180 の円」と学びました。**赤は、その円のつなぎ目（0 と 180 の両方）にまたがっています**。そのため「赤を抜き出したい」ときだけは、`cv2.inRange` を1回では指定できず、0付近の範囲と180付近の範囲を**2回に分けて**抜き出し、あとで合体させる必要があります。

`leaf_monitor.py` の `get_mat_mask()` は、この赤の特殊事情にも対応できるよう作られています。ただし**青や緑のマットを使う分には、ふつうの1回の `inRange` で済みます**。今回は深入りせず、「赤マットを使うと少しややこしい」とだけ覚えておけば十分です（だから仕様書でも青・緑が推奨されています）。

---

## 3.6 `leaf_monitor.py` への橋渡し

この章の主役 `cv2.inRange` は、`leaf_monitor.py` の `get_mat_mask()` の中心です。

```python
return cv2.inRange(hsv, lower.astype(np.uint8), upper.astype(np.uint8))
```

私たちが `step8.py` でやったのと同じく、HSV 画像から「マット色の範囲」を白く抜き出しています。

そして `lower` と `upper` がどう決まるかというと、`calibrate()` の中で「マットの代表色 ± マージン」で計算されています。

```python
lower = [peak - H_MARGIN, max(0, s_med - S_MARGIN), max(0, v_med - V_MARGIN)]
upper = [peak + H_MARGIN, min(255, s_med + S_MARGIN), min(255, v_med + V_MARGIN)]
```

`step10.py` で私たちが手作業で組み立てた式と、まったく同じ形をしています。`peak`（マットの代表的な色相）は、第2章でふれた「色相ヒストグラムの最頻値」、つまり**一番面積を占める色＝マットの色**です。`H_MARGIN`・`S_MARGIN`・`V_MARGIN` は、いま「やってみよう」で動かしてみた、あの許容幅そのものです。

> なお `leaf_monitor.py` では、`inRange` の前に S や V の最低・最高ライン（`SAT_MIN`・`VAL_MIN`・`VAL_MAX`）も使って、灰色・黒・白飛びした画素を「色なし」として代表色の計算から外しています。これも「マットの色味だけを正しくつかむ」ための工夫です。

- `cv2.inRange(hsv, 下限, 上限)` で、範囲内の画素を白くしたマスクを作れる。中身は 0 と 255 だけ。

- 赤はH円のつなぎ目にまたがるので少し特殊。青・緑マットなら気にしなくてよい。

# 第4章 マスクの掃除（モルフォロジー）

> 第3章で作ったマスクをよく見ると、たいてい「汚れて」います。
> 小さな白い点がぱらぱら散っていたり、白い領域の中に黒い穴があいていたり。
> この章では、マスクをきれいに整える画像処理 **モルフォロジー** を学びます。地味ですが、計測の安定にとても効きます。

---

## 4.1 この章のゴール

この章を終えると、次のことができるようになります。

- マスクに混じる「小さなノイズ」「小さな穴」がなぜ問題になるかを説明できる
- `cv2.getStructuringElement` で、処理に使う「ブラシ」の形を作れる
- **オープニング**（小さな白い点を消す）と**クロージング**（小さな黒い穴を埋める）を使い分けられる
- **膨張（dilate）** で白い領域を少し太らせられる
- `leaf_monitor.py` がマスクをどう整えているか理解できる

---

## 4.2 なぜマスクを掃除するのか

第3章で作った `mat_mask.jpg` をよく見ると、たとえばこんな汚れがあります。

- マットの外の関係ない場所に、**小さな白い点**がぽつぽつ（似た色のゴミや、光の反射を拾ってしまった）
- マットの白い領域の中に、**小さな黒い穴**（落ち葉の影や模様で、一部がマット色から外れた）

このまま「白い部分の数」や「白い領域の面積」を数えると、ノイズのぶんだけ結果がブレてしまいます。`leaf_monitor.py` は1日に何度も自動で計測するので、こうした小さなブレが積み重なると、トレンドが読みにくくなります。

そこで、**計測の前にマスクを掃除して形を整える**のです。この掃除の技術が「モルフォロジー処理」です。

---

## 4.3 「ブラシ」を用意する：getStructuringElement

モルフォロジー処理は、ある形の「ブラシ（カーネル）」でマスクをなぞるように働きます。まずそのブラシを作ります。

```python
import cv2
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
```

- `cv2.MORPH_ELLIPSE` … ブラシの形を「楕円（だ円）」にする。角がなく自然な仕上がりになるので、`leaf_monitor.py` でも使われています。
- `(5, 5)` … ブラシの大きさ（5×5ピクセル）。**大きいほど効き目が強く**、小さな汚れだけでなく大きめのものまで影響します。

このブラシの大きさが、`leaf_monitor.py` の `MORPH_KERNEL = 5` という設定にあたります。

---

## 4.4 オープニング：小さな白い点を消す

**オープニング**は、「小さな白い出っぱりやノイズを消す」処理です。ブラシより小さい白い点は消え、本体の大きな白い領域はほぼそのまま残ります。

第3章で作った `mat_mask.jpg` を掃除してみましょう。

```python
# step11.py
import cv2

mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)   # マスクは白黒で読む
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))

opened = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel, iterations=1)

cv2.imwrite("mask_opened.jpg", opened)
print("mask_opened.jpg を保存しました。元の mat_mask.jpg と見比べてください。")
```

`mat_mask.jpg` と `mask_opened.jpg` を VS Code で並べて開いてください。背景に散っていた**小さな白い点が消えて**、すっきりしているはずです。

> ポイント：`cv2.imread(..., cv2.IMREAD_GRAYSCALE)` でマスクを白黒のまま読み込んでいます。マスクはもともと白黒なので、3つ組のカラーで読む必要はありません。`leaf_monitor.py` も、保存したマスク（region.png など）をこの白黒モードで読み込んでいます。

> **やってみよう 1**
> `kernel` の大きさを `(5, 5)` から `(15, 15)` に変えて実行してみましょう。ブラシが大きくなると、消える白い部分も大きくなります。やりすぎると本来のマットまで削れてしまうことを確認してください。

---

## 4.5 クロージング：小さな黒い穴を埋める

**クロージング**は、オープニングの逆向きで、「小さな黒い穴やすき間を埋める」処理です。白い領域の中の小さな黒い点が白く埋まります。

```python
# step12.py
import cv2

mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))

closed = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel, iterations=2)

cv2.imwrite("mask_closed.jpg", closed)
print("mask_closed.jpg を保存しました。穴が埋まったか確認してください。")
```

`MORPH_CLOSE` を指定し、`iterations=2`（2回くりかえし）で少し強めに効かせています。白い領域の中にあった小さな黒い穴が**埋まって**、なめらかになっているはずです。

オープニングとクロージングは、しばしば**セット**で使われます。「まずオープニングでノイズを消し、次にクロージングで穴を埋める」と、白い領域がきれいなひと固まりになります。`leaf_monitor.py` でも、落ち葉のマスクに対してこの順で両方かけています。

> **やってみよう 2**
> 「オープニングだけ」「クロージングだけ」「オープニング→クロージングの順」の3つを作って見比べてみましょう。3つ目が一番きれいになるはずです。なぜ順番が大事なのか考えてみてください。

---

## 4.6 膨張（dilate）：白い領域を少し太らせる

**膨張（dilate）** は、白い領域を全体的にひとまわり太らせる処理です。

これは何に使うのでしょうか。`leaf_monitor.py` では、「鉢」を計測から除外するためのマスクを作るとき、鉢のマスクを少し太らせています。なぜなら、抜き出した鉢の輪郭はぴったりすぎて、**鉢の縁がわずかに取りこぼされて堆積物と誤検出される**ことがあるからです。少し太らせて余裕を持たせることで、縁の取りこぼしを防ぎます。

```python
# step13.py
import cv2

mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (9, 9))

dilated = cv2.dilate(mask, kernel, iterations=1)

cv2.imwrite("mask_dilated.jpg", dilated)
print("mask_dilated.jpg を保存しました。白い部分が太ったか確認してください。")
```

`mat_mask.jpg` と `mask_dilated.jpg` を比べると、白い領域がひとまわり**膨らんで**いるはずです。ここで使った `(9, 9)` というブラシの大きさが、`leaf_monitor.py` の `POT_DILATE = 9`（鉢マスクをどれだけ太らせるか）にあたります。

> **やってみよう 3**
> 仕様書のトラブルシューティングに「鉢の縁が堆積物として残るときは `POT_DILATE` を大きくする」とあります。`(9, 9)` を `(21, 21)` にすると白い部分はもっと太ります。ただし太らせすぎると、鉢の近くにある本物の堆積物まで「鉢」として消してしまいます。このトレードオフを確認してください。

---

## 4.7 `leaf_monitor.py` への橋渡し

この章で学んだ処理は、`leaf_monitor.py` のあちこちに出てきます。

**(1) ブラシを作る** — `analyze()`・`calibrate()` の両方

```python
k = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (MORPH_KERNEL, MORPH_KERNEL))
```

私たちの `kernel` と同じ、楕円のブラシです。`MORPH_KERNEL` が大きさ（既定値5）です。

**(2) ノイズ消し＋穴埋め** — `analyze()` の落ち葉マスク処理

```python
leaf_mask = cv2.morphologyEx(leaf_mask, cv2.MORPH_OPEN,  k, iterations=1)
leaf_mask = cv2.morphologyEx(leaf_mask, cv2.MORPH_CLOSE, k, iterations=2)
```

まさに「オープニングでノイズを消し、クロージングで穴を埋める」をこの順で行っています。`step11.py`〜`step12.py` でやったことそのものです。

**(3) 鉢マスクを太らせる** — `calibrate()` の中

```python
dk = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (POT_DILATE, POT_DILATE))
pot = cv2.dilate(pot, dk, iterations=1)
```

`step13.py` と同じく、鉢マスクを少し太らせて縁の取りこぼしを防いでいます。`POT_DILATE = 9` がブラシの大きさです。

このように、第3章で作った「ざらついたマスク」を、この章のモルフォロジーで「なめらかなマスク」に整えてから、次のステップに渡しています。計測の安定は、こうした地道な掃除に支えられているのです。

---

## 4.8 この章のまとめ

- マスクには小さな白い点（ノイズ）や黒い穴が混じりがちで、そのまま数えると結果がブレる。
- `cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (k, k))` で処理用のブラシ（カーネル）を作る。大きいほど効き目が強い。
- **オープニング（`MORPH_OPEN`）** … 小さな白い点を消す。
- **クロージング（`MORPH_CLOSE`）** … 小さな黒い穴を埋める。オープニングとセットで使うときれい。
- **膨張（`cv2.dilate`）** … 白い領域を太らせる。鉢の縁の取りこぼし防止に使う（`POT_DILATE`）。

# 第5章 マスクを組み合わせる（ビット演算）

> ここまでで、きれいなマスクが作れるようになりました。
> この章では複数のマスクを「かつ」「でない」で組み合わせて、**「計測範囲の中で・マット色でないもの＝堆積物」** を取り出します。
> `leaf_monitor.py` が落ち葉を抜き出す、いちばん核心の1行を読み解く章です。

---

## 5.1 この章のゴール

この章を終えると、次のことができるようになります。

- マスク同士を「論理」で組み合わせるという発想を理解できる
- `cv2.bitwise_not`（白黒を反転＝「でない」）が使える
- `cv2.bitwise_and`（両方が白の所だけ残す＝「かつ」）が使える
- 「範囲の中で・マット色でない部分＝堆積物」を、自分で組み立てられる
- `leaf_monitor.py` の落ち葉抽出の核心を読み解ける

---

## 5.2 マスクは「論理」で組み合わせられる

マスクは白（255）と黒（0）だけの画像でした。白を「はい（真）」、黒を「いいえ（偽）」と考えると、マスク同士を**論理**で組み合わせられます。

- **NOT（でない）**：白と黒をひっくり返す。「マット色」→「マット色でない」。
- **AND（かつ）**：両方が白の場所だけ白く残す。「範囲の中」かつ「マット色でない」。
- （OR（または）：どちらかが白なら白。今回の主役ではありませんが、赤マットの2範囲の合体などで使われます。）

私たちがやりたいのは、まさにこれです。

> **堆積物 ＝ 「計測範囲の中」 かつ 「マット色でない」**

この日本語の文を、そのまま `AND` と `NOT` で組み立てれば、堆積物のマスクができあがります。

---

## 5.3 NOT：マスクを反転する

まず「マット色でない」を作ります。第3章で作った「マット色のマスク」を反転すればよいですね。`cv2.bitwise_not` を使います。

```python
# step14.py
import cv2

mat_mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)   # マット色＝白

not_mat = cv2.bitwise_not(mat_mask)   # 白黒を反転 ＝ マット色でない＝白

cv2.imwrite("not_mat.jpg", not_mat)
print("not_mat.jpg を保存しました。白黒が反転したか確認してください。")
```

`mat_mask.jpg` と `not_mat.jpg` を見比べてください。**白と黒が完全に入れ替わって**います。`not_mat.jpg` では、マット以外（鉢・落ち葉・背景）が白くなっているはずです。

ただし、この `not_mat.jpg` には鉢も背景も全部入ってしまっています。私たちが欲しいのは「**計測範囲の中の**マット色でない部分」だけ。そこで次の AND で範囲を絞ります。

---

## 5.4 AND：2つのマスクの「重なり」だけ残す

`cv2.bitwise_and` は、2つのマスクの**両方が白の場所だけ**を白く残します。「重なり（共通部分）」を取り出すイメージです。

「計測範囲」のマスクが必要ですが、これは本来は第6章でちゃんと作ります。この章では練習として、画像の中央の長方形を「仮の計測範囲」として手作りして、仕組みを体験しましょう。

```python
# step15.py
import cv2
import numpy as np

mat_mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)
h, w = mat_mask.shape

# 仮の計測範囲：中央の大きな長方形を白く塗ったマスク（本番は第6章で作る）
region = np.zeros((h, w), dtype=np.uint8)
cv2.rectangle(region, (w//5, h//5), (w*4//5, h*4//5), 255, -1)  # -1で内部を塗りつぶし

# 「マット色でない」
not_mat = cv2.bitwise_not(mat_mask)

# 「範囲の中」かつ「マット色でない」＝堆積物
leaf = cv2.bitwise_and(region, not_mat)

cv2.imwrite("region.jpg", region)
cv2.imwrite("leaf.jpg", leaf)
print("region.jpg（仮の範囲）と leaf.jpg（取り出した堆積物）を保存しました。")
```

`region.jpg`・`leaf.jpg` を VS Code で開いてください。

- `region.jpg` は、中央の長方形だけが白い「仮の計測範囲」です。

## 5.5 同じ考え方で「鉢」も取り出せる

この AND と NOT の組み合わせは、`leaf_monitor.py` のキャリブレーション（最初の準備）でも大活躍します。

落ち葉がない状態で撮った写真では、「マットの範囲の中で・マット色でないもの」は何でしょうか? 落ち葉がないのですから、それは**鉢などの固定物**です。つまり、

> 鉢 ＝ 「マット範囲の中」 かつ 「マット色でない」

これも同じ式で書けます。落ち葉がある計測時には「マット色でないもの＝堆積物」、落ち葉がないキャリブレーション時には「マット色でないもの＝鉢」。同じ仕組みを、状況によって違う目的に使い回しているわけです。とても賢い設計です。

## 5.6 `leaf_monitor.py` への橋渡し

この章の核心は、`analyze()` の中のこの1行に凝縮されています。

```python
leaf_mask = cv2.bitwise_and(region, cv2.bitwise_not(mat_color))
```

`step15.py` で私たちが書いた `cv2.bitwise_and(region, not_mat)` と、**まったく同じ形**です。

- `region` … 固定された計測範囲のマスク（第6章・第7章で詳しく扱います）
- `cv2.bitwise_not(mat_color)` … 「マット色でない」（私たちの `not_mat`）
- `bitwise_and(...)` … 「範囲の中」かつ「マット色でない」＝堆積物

この1行で「堆積物の場所」が確定します。`leaf_monitor.py` のいちばん肝心な部分が、第3章のマスク・第4章の掃除・この章のビット演算という、ここまで学んだ3つの組み合わせでできていることが分かります。

そしてキャリブレーション側では、`calibrate()` に次の行があります。

```python
pot = cv2.bitwise_and(mat_region, cv2.bitwise_not(mat_color))
```

5.5 で話した「鉢 ＝ マット範囲 かつ マット色でない」が、この1行です。同じ道具で鉢を取り出しているのが見て取れます。

## 5.7 この章のまとめ

- マスクの白を「真」、黒を「偽」と考えると、マスク同士を**論理**で組み合わせられる。
- `cv2.bitwise_not(mask)` … 白黒を反転（「〜でない」）。「マット色」→「マット色でない」。
- `cv2.bitwise_and(a, b)` … 両方が白の場所だけ残す（「かつ」）。範囲の絞り込みに使う。
- 「**範囲の中** かつ **マット色でない**＝堆積物」が、そのまま `bitwise_and(region, bitwise_not(mat))` になる。
- 同じ式が、計測時は「堆積物」、キャリブレーション時は「鉢」を取り出すのに使い回される。

# 第6章 形を見つける（輪郭・凸包）

> 堆積物やマットの「場所」はマスクで分かりました。
> でも「塊がいくつあるか」「マット全体の外形はどこか」を知るには、白い領域の**ふちどり（輪郭）**を見つける必要があります。
> この章では輪郭まわりの道具を学び、`leaf_monitor.py` が鉢で分断されたマットを1つに復元する、賢い仕組みを解き明かします。

---

## 6.1 この章のゴール

この章を終えると、次のことができるようになります。

- **輪郭（contour）** とは「白い領域のふちどり」だと説明できる
- `cv2.findContours` で輪郭を取り出し、`cv2.contourArea` で面積を測れる
- 面積で小さな輪郭（ノイズ）をふるい落とす方法が分かる
- `cv2.convexHull`（凸包）で、バラバラの断片を1つの外形にまとめられる
- `cv2.fillPoly` で、その外形を塗りつぶしてマスクにできる

---

## 6.2 輪郭（contour）ってなに？

**輪郭**は、白い領域の「ふちどり（境界線）」のことです。塗り絵で言えば、色を塗る前の「黒い線」にあたります。

輪郭が手に入ると、次のようなことができます。

- 白い塊が**いくつ**あるか数える（輪郭の本数＝塊の数）
- それぞれの塊の**面積**を測る
- 塊を線で**囲んで**確認画像を作る（第8章）

`leaf_monitor.py` は、これを使って「堆積物の塊が何個あるか（clumps）」を数えたり、「マットの外形」を求めたりしています。

---

## 6.3 findContours で輪郭を取り出す

第5章で作った堆積物マスク `leaf.jpg` から、輪郭を取り出してみましょう。

```python
# step16.py
import cv2

leaf = cv2.imread("leaf.jpg", cv2.IMREAD_GRAYSCALE)

contours, _ = cv2.findContours(leaf, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

print("見つかった輪郭の数:", len(contours))
```

```bash
python3 step16.py
```

`cv2.findContours` の3つの引数・戻り値を説明します。

- 第1引数：輪郭を探したい白黒マスク。
- `cv2.RETR_EXTERNAL`：**いちばん外側の輪郭だけ**を取る、という指定。塊の中に穴があっても、外側の囲いだけを取ります。塊を数えるのにちょうどよい設定です。
- `cv2.CHAIN_APPROX_SIMPLE`：輪郭を表す点を**間引いて軽く**する指定。たとえばまっすぐな辺は両端の2点だけで表します。データが小さくなり速くなります。
- 戻り値：`contours`（輪郭のリスト）と、もう1つ（輪郭の親子関係。今回は使わないので `_` で受け捨て）。

`len(contours)` が、見つかった白い塊のおおよその数です。ただし、この中には影やノイズによる**ごく小さな塊**も混じっています。次でふるい落とします。

> `cv2.imread(..., cv2.IMREAD_GRAYSCALE)` でマスクを白黒のまま読み込んでいる点に注目してください。輪郭処理は白黒マスクに対して行います。

---

## 6.4 contourArea で面積を測り、小さいものを捨てる

`cv2.contourArea` は、1つの輪郭が囲む**面積（ピクセル数）**を返します。これを使って、「小さすぎる塊はノイズとして数えない」というふるい分けをします。

```python
# step17.py
import cv2

leaf = cv2.imread("leaf.jpg", cv2.IMREAD_GRAYSCALE)
contours, _ = cv2.findContours(leaf, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

MIN_LEAF_AREA = 600   # これ未満は影・ノイズとみなして数えない

clumps = [c for c in contours if cv2.contourArea(c) >= MIN_LEAF_AREA]

print("全部の輪郭:", len(contours))
print("面積でふるった後の塊:", len(clumps))
```

`MIN_LEAF_AREA` 以上の面積を持つ輪郭だけを `clumps` に残しています。実行すると、ふるった後の数のほうが少なくなるはずです。小さなゴミ・影を除いた、本物の塊の数が `len(clumps)` です。

> **やってみよう 1**
> `MIN_LEAF_AREA` を `600 → 50` に下げると、ふるった後の塊の数はどうなるでしょうか? 予想してから実行してください。しきい値を下げるほど、小さなノイズまで「塊」として数えてしまうことを確認しましょう。

この `MIN_LEAF_AREA` が、`leaf_monitor.py` の同名の設定です。塊数（clumps）の数え方を決める、大事なつまみです。

## 6.5 凸包（convexHull）：バラバラを1つにまとめる

ここが `leaf_monitor.py` のいちばん賢いところの1つです。

マットの上には鉢が置かれています。すると、鉢に隠れてマットが**いくつかの断片に分断**されて見えることがあります。マット色のマスクを取ると、1枚のマットなのに、左・右・上…と複数の白い島に分かれてしまうのです。

でも私たちが知りたいのは「マット全体の外形」です。バラバラの断片を、**1つの大きな外形にまとめたい**。そこで使うのが **凸包（convex hull・とつぼう）** です。

凸包は、「散らばった点すべてを、外側からピンと張った輪ゴムで囲んだときの形」だと思ってください。へこみのない、全体を包む最小の多角形ができます。

```python
# step18.py
import cv2
import numpy as np

mat = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)
contours, _ = cv2.findContours(mat, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

# 十分大きい断片だけ集める（小さなゴミは無視）
MIN_MAT_FRAGMENT = 2000
pts = [c for c in contours if cv2.contourArea(c) >= MIN_MAT_FRAGMENT]

# 断片の点を全部まとめて、外側を輪ゴムで囲う＝凸包
hull = cv2.convexHull(np.vstack(pts))

print("断片の数:", len(pts))
print("凸包の頂点の数:", len(hull))
```

`np.vstack(pts)` は、複数の断片の点を**縦に積み重ねて1つにまとめる**NumPy の操作です。そうしてまとめた全部の点を `cv2.convexHull` に渡すと、それらを囲む1つの外形が返ってきます。これで「分断されたマット」が「1つのマット外形」に復元されました。

`MIN_MAT_FRAGMENT`（既定2000）は、「これより小さい白い島は、マットの断片とは認めず無視する」というしきい値です。小さな緑のゴミなどを、うっかりマットに含めないための予防線です。

## 6.6 fillPoly：外形を塗りつぶしてマスクにする

凸包はまだ「ふちどり（線）」です。これを計測に使うには、内部を白く塗りつぶして**マスク**にする必要があります。`cv2.fillPoly` を使います。

```python
# step19.py
import cv2
import numpy as np

mat = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)
contours, _ = cv2.findContours(mat, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
pts = [c for c in contours if cv2.contourArea(c) >= 2000]
hull = cv2.convexHull(np.vstack(pts))

# 真っ黒な画像を用意し、凸包の内部を白(255)で塗る
region = np.zeros(mat.shape, dtype=np.uint8)
cv2.fillPoly(region, [hull], 255)

cv2.imwrite("mat_region.jpg", region)
print("mat_region.jpg を保存しました。マット全体が白い1枚の領域になっているか確認してください。")
```

`np.zeros(mat.shape, dtype=np.uint8)` で、まず**真っ黒な台紙**を用意します（第0章のおさらいですね）。そこに `cv2.fillPoly` で凸包の内部を白く塗ります。

`mat_region.jpg` を VS Code で開くと、鉢で分断されていたマットが、**穴のない1枚の白い領域**になっているはずです。これが「マット全体の外形」のマスクで、`leaf_monitor.py` の `region.png` のもとになります。

> `mat_mask.jpg`（分断されている）と `mat_region.jpg`（凸包で埋めた）を見比べてください。凸包が「断片のすき間」までまとめて1つにしていることが分かります。この後、ここから鉢を除けば、いよいよ計測範囲が完成します（第7章）。

## 6.7 `leaf_monitor.py` への橋渡し

この章の道具は、`mat_region_from_mask()` という関数にまとまって登場します。

# 第7章 面積を測る（被覆面積率）

> いよいよ、このプログラムの主役の数字「**被覆面積率（％）**」を計算します。
> 仕組みは拍子抜けするほど素朴で、「白い画素を数えて、割り算する」だけ。
> でもその裏には「分母を固定する」という、トレンド監視のための大事な工夫が隠れています。

---

## 7.1 この章のゴール

この章を終えると、次のことができるようになります。

- `cv2.countNonZero` で、マスクの白い画素の数（＝面積）を数えられる
- 「被覆面積率＝堆積物の面積 ÷ 計測範囲の面積 × 100」を計算できる
- なぜ計測範囲（分母）を**固定**しなければならないかを説明できる
- マスクをファイルに保存し、`IMREAD_GRAYSCALE` で読み戻す流れを理解できる

---

## 7.2 面積を数える：countNonZero

マスクは白（255）と黒（0）だけの画像でした。白い画素1つを「1ピクセルの面積」と考えれば、**白い画素の数を数えれば、それがそのまま面積**になります。

`cv2.countNonZero` は、その名のとおり「0でない画素（＝白い画素）の数」を数えてくれる関数です。

```python
# step20.py
import cv2

region = cv2.imread("mat_region.jpg", cv2.IMREAD_GRAYSCALE)   # 第6章で作ったマット外形
leaf   = cv2.imread("leaf.jpg",       cv2.IMREAD_GRAYSCALE)   # 第5章で作った堆積物

region_area = cv2.countNonZero(region)
leaf_area   = cv2.countNonZero(leaf)

print("計測範囲の面積:", region_area, "px")
print("堆積物の面積  :", leaf_area,   "px")
```

```bash
python3 step20.py
```

それぞれの白い画素数（ピクセル単位の面積）が表示されます。これで「全体の広さ」と「覆われている広さ」の両方が手に入りました。

---

## 7.3 割り算して「被覆面積率」を出す

あとは割り算です。「覆われている面積」を「全体の面積」で割り、100 を掛ければパーセントになります。

```python
# step21.py
import cv2

region = cv2.imread("mat_region.jpg", cv2.IMREAD_GRAYSCALE)
leaf   = cv2.imread("leaf.jpg",       cv2.IMREAD_GRAYSCALE)

region_area = cv2.countNonZero(region)
leaf_area   = cv2.countNonZero(leaf)

coverage_pct = 100.0 * leaf_area / region_area if region_area else 0.0
print(f"被覆面積率 = {coverage_pct:.2f}%")
```

`if region_area else 0.0` は、「範囲の面積が0のときに0で割ってエラーになるのを防ぐ」ための安全装置です（割り算の前に0でないか確認している）。

これで、`leaf_monitor.py` が CSV に記録している主役の数字「`coverage_pct`」が、自分の手で計算できました。仕組みは「**白い画素を数えて割る**」だけ。とても素朴ですね。

> **やってみよう 1**
> マットの上に落ち葉を数枚足して、もう1枚撮影し（`rpicam-jpeg`）、第3章〜第5章と同じ手順で堆積物マスクを作り直してから、もう一度この計算をしてみましょう。落ち葉が増えたぶん、`coverage_pct` が**上がる**はずです。これがまさに「増加トレンドを追う」ということです。

## 7.4 いちばん大事な工夫：分母を固定する

ここで、`leaf_monitor.py` の設計の核心に触れます。仕様書「3.6 計測範囲：キャリブレーション時に固定」で説明されている、重要な考え方です。

被覆面積率は「堆積物の面積 ÷ **計測範囲の面積**」でした。この**分母（計測範囲の面積）が日によって変わってしまうと、値を正しく比べられません**。

なぜ変わりうるのでしょうか。もし毎回その日の写真からマット外形を計算し直すと、堆積物が増えてマットが隠れていくにつれ、見えるマットがどんどん小さくなります。すると分母も小さくなり、「分母も分子も動く」状態になって、トレンドが崩れてしまいます。

そこで `leaf_monitor.py` は、**マットがきれいな最初の状態で計測範囲を1回だけ確定し、ファイル（`region.png`）に保存して固定**します。以降の計測では、毎回この同じ範囲を分母に使います。

> 分母が毎回同じ ＝ 「ものさし」が変わらない。だから日々の被覆率の増減が、そのまま堆積物の増減を表す。

これが、トレンド監視を正しく行うためのいちばんの肝です。私たちが `step20.py`〜`step21.py` で `mat_region.jpg` を毎回読み込んで使ったのも、同じ「決まったものさしを使い回す」発想です。

---

## 7.5 マスクを保存して、読み戻す

「固定する」を実現するには、計測範囲のマスクをファイルに保存しておき、計測のたびに読み戻す必要があります。第1章で学んだ `cv2.imwrite` と `cv2.imread` の出番です。

- 保存：`cv2.imwrite("region.png", region)` … キャリブレーション時に1回だけ。
- 読み戻し：`cv2.imread("region.png", cv2.IMREAD_GRAYSCALE)` … 計測のたびに。

ここで `cv2.IMREAD_GRAYSCALE` を付けているのがポイントです。マスクは白黒なので、カラー（3つ組）ではなく**白黒（1マス1個）で読み込む**よう指定しています。こうしておけば、`countNonZero` などの白黒マスク向けの処理にそのまま渡せます。

> **やってみよう 2**
> `cv2.imwrite("region.png", region)` で保存し、別のスクリプトから `cv2.imread("region.png", cv2.IMREAD_GRAYSCALE)` で読み戻して、`countNonZero` の値が保存前と同じになることを確認しましょう。「一度決めた範囲を、別の実行でも同じものさしとして使える」ことが体感できます。

## 8.3 drawContours で輪郭を描く

`cv2.drawContours` は、輪郭（ふちどり）を画像の上に線で描く関数です。第6章で取り出した輪郭を、もとの写真に重ねてみましょう。

```python
# step22.py
import cv2

img  = cv2.imread("mat.jpg")                                  # もとのカラー写真
leaf = cv2.imread("leaf.jpg", cv2.IMREAD_GRAYSCALE)           # 堆積物マスク

contours, _ = cv2.findContours(leaf, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

annotated = img.copy()                                        # もとを壊さないようコピーに描く
cv2.drawContours(annotated, contours, -1, (0, 255, 0), 2)     # 緑で全部の輪郭を描く

cv2.imwrite("annotated_step.jpg", annotated)
print("annotated_step.jpg を保存しました。落ち葉が緑で囲まれているか確認してください。")
```

`cv2.drawContours` の引数を見ていきます。

- `annotated` … 描き込む先の画像（もとの写真のコピー）。
- `contours` … 描きたい輪郭のリスト。
- `-1` … 「リストの**全部**の輪郭を描く」という意味（特定の1本だけ描きたいなら 0, 1, … と番号を指定）。
- `(0, 255, 0)` … 線の色。**ここが要注意です**（次の節）。
- `2` … 線の太さ（ピクセル）。

> `img.copy()` でコピーに描いているのは、もとの `img` を壊さないためです。元画像はそのまま取っておきたいので、描画は必ずコピーに対して行います。

---

## 8.4 ふたたびBGRの罠：色の順番

`(0, 255, 0)` が緑になるのはなぜでしょうか。第1章を思い出してください。**OpenCV の色は BGR（青・緑・赤）の順**でした。色の指定もこの順番です。

- `(255, 0, 0)` → **青**（Blueが255）
- `(0, 255, 0)` → **緑**（Greenが255）
- `(0, 0, 255)` → **赤**（Redが255）
- `(0, 255, 255)` → **黄**（GreenとRedが255。緑＋赤＝黄）

「赤を描きたくて `(255, 0, 0)` と書いたら青になった」——これは初心者が必ず一度はやる失敗です。第1章で学んだ BGR の順番が、描画でもそのまま効いてくるわけです。

`leaf_monitor.py` の色分けも、この BGR で決まっています。

- 計測範囲：`(255, 0, 0)` ＝ **青**
- 鉢（除外）：`(0, 255, 255)` ＝ **黄**
- 堆積物：`(0, 255, 0)` ＝ **緑**

> **やってみよう 1**
> `step22.py` の `(0, 255, 0)` を `(0, 0, 255)` に変えて実行し、落ち葉の囲み線が**赤**になることを確認しましょう。次に `(255, 0, 0)` にすると青になります。BGR の順番を体で覚えてください。

## 8.5 putText で数値を書き込む

最後に、被覆面積率と塊数を画像の中に文字で書き込みます。`cv2.putText` を使います。

```python
# step23.py
import cv2

img  = cv2.imread("mat.jpg")
leaf = cv2.imread("leaf.jpg", cv2.IMREAD_GRAYSCALE)
region = cv2.imread("mat_region.jpg", cv2.IMREAD_GRAYSCALE)

contours, _ = cv2.findContours(leaf, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
clumps = [c for c in contours if cv2.contourArea(c) >= 600]

region_area = cv2.countNonZero(region)
leaf_area   = cv2.countNonZero(leaf)
coverage = 100.0 * leaf_area / region_area if region_area else 0.0

annotated = img.copy()
cv2.drawContours(annotated, contours, -1, (0, 255, 0), 2)   # 堆積物：緑

label = f"coverage={coverage:.2f}%  clumps={len(clumps)}"
cv2.putText(annotated, label, (10, 30),
            cv2.FONT_HERSHEY_SIMPLEX, 1.0, (0, 0, 255), 2)   # 赤い文字

cv2.imwrite("annotated_full.jpg", annotated)
print("annotated_full.jpg を保存しました。左上に数値が出ているか確認してください。")
```

`cv2.putText` の引数です。

- `annotated` … 書き込む先の画像。
- `label` … 書き込む文字列。
- `(10, 30)` … 文字を書き始める位置（左から10、上から30）。
- `cv2.FONT_HERSHEY_SIMPLEX` … フォントの種類（OpenCV に用意された定番フォント）。
- `1.0` … 文字の大きさ。
- `(0, 0, 255)` … 文字の色。BGR なので、これは**赤**です。
- `2` … 文字の線の太さ。

`annotated_full.jpg` を VS Code で開いてください。落ち葉が緑で囲まれ、左上に `coverage=...%  clumps=...` と赤い文字で表示された、**第0章で予告した確認画像**ができあがっているはずです。ここがこのテキストの1つのゴール地点です。

## 9.4 キャリブレーションを読む（calibrate）

`calibrate()` を、コードの流れに沿って読みます。

**① 撮影して読み込む（第1章）**

```python
capture_image(path)
img = cv2.imread(path)
```

落ち葉のない状態を撮り、数字の表として読み込みます。

**② HSV に変換し、3つに分ける（第2章）**

```python
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
Hc, Sc, Vc = hsv[:, :, 0], hsv[:, :, 1], hsv[:, :, 2]
```

色味で判断するため HSV にし、H・S・V を取り出します。このあと、最も面積を占める色相（H）を「マットの色」と判断します。

**③ マットの色範囲を決めてマスクを作る（第3章）**

```python
lower = [peak - H_MARGIN, ...]
upper = [peak + H_MARGIN, ...]
mat_color = get_mat_mask(hsv, np.array(lower), np.array(upper))   # 中で cv2.inRange
```

「代表色 ± マージン」で範囲を決め、`cv2.inRange` でマット色のマスクを作ります。

**④ マスクを掃除する（第4章）**

```python
k = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (MORPH_KERNEL, MORPH_KERNEL))
mat_color = cv2.morphologyEx(mat_color, cv2.MORPH_OPEN, k, iterations=1)
```

オープニングで小さなノイズを消します。

**⑤ マット外形を凸包でまとめる（第6章）**

```python
mat_region = mat_region_from_mask(mat_color)   # 中で findContours→contourArea→convexHull→fillPoly
```

鉢で分断されたマットを、1つの外形マスクに復元します。

**⑥ 鉢を取り出して太らせる（第5章・第4章）**

```python
pot = cv2.bitwise_and(mat_region, cv2.bitwise_not(mat_color))   # マット範囲 かつ マット色でない＝鉢
pot = cv2.morphologyEx(pot, cv2.MORPH_OPEN, k, iterations=2)
pot = cv2.dilate(pot, dk, iterations=1)                          # 縁の取りこぼし対策
cv2.imwrite(POT_MASK, pot)
```

落ち葉がないので、「マット範囲の中でマット色でないもの」は鉢です。少し太らせて保存します。

**⑦ 計測範囲を確定して固定保存（第5章・第7章）**

```python
region = cv2.bitwise_and(mat_region, cv2.bitwise_not(pot))   # マット範囲 から 鉢 を除く
cv2.imwrite(REGION_MASK, region)
```

マット外形から鉢を除いた領域が、固定の計測範囲です。`region.png` に保存し、以降のものさしにします。

これでキャリブレーション完了。`mat_config.json`（色範囲）・`pot.png`（鉢）・`region.png`（計測範囲）の3つが用意されました。

## 9.5 計測を読む（run_once → analyze）

次に、ふだんの計測 `run_once()` と、その中で呼ばれる `analyze()` を読みます。

**① 固定設定を読み戻す（第7章）**

```python
region = cv2.imread(REGION_MASK, cv2.IMREAD_GRAYSCALE)
pot = cv2.imread(POT_MASK, cv2.IMREAD_GRAYSCALE) if os.path.exists(POT_MASK) else None
```

キャリブレーションで保存した固定の計測範囲を、白黒で読み戻します。**毎回同じものさし**を使うのが肝でした。

**② 撮影して読み込む（第1章）**

```python
capture_image(cap_path)
img = cv2.imread(cap_path)
```

**③ HSV にしてマット色マスクを作り、掃除する（第2〜4章）**

```python
hsv = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2HSV)
mat_color = get_mat_mask(hsv, lower, upper)              # inRange
mat_color = cv2.morphologyEx(mat_color, cv2.MORPH_OPEN, k, iterations=1)
```

**④ 範囲の中でマット色でないもの＝堆積物を取り出す（第5章・第4章）**

```python
leaf_mask = cv2.bitwise_and(region, cv2.bitwise_not(mat_color))   # 範囲 かつ マット色でない
leaf_mask = cv2.morphologyEx(leaf_mask, cv2.MORPH_OPEN,  k, iterations=1)
leaf_mask = cv2.morphologyEx(leaf_mask, cv2.MORPH_CLOSE, k, iterations=2)
```

このテキストでいちばん大事にしてきた1行です。「範囲の中・マット色でない＝堆積物」。

**⑤ 面積を数えて被覆率を計算（第7章）**

```python
region_area = int(cv2.countNonZero(region))   # 固定なので毎回同じ
leaf_area   = int(cv2.countNonZero(leaf_mask))
coverage_pct = 100.0 * leaf_area / region_area if region_area else 0.0
```

**⑥ 塊を数える（第6章）**

```python
leaf_cnts, _ = cv2.findContours(leaf_mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
clumps = [c for c in leaf_cnts if cv2.contourArea(c) >= MIN_LEAF_AREA]
```

**⑦ 確認画像を描く（第8章）**

```python
cv2.drawContours(annotated, rcnts,  -1, (255, 0, 0), 2)   # 範囲: 青
cv2.drawContours(annotated, pcnts,  -1, (0, 255, 255), 2) # 鉢: 黄
cv2.drawContours(annotated, clumps, -1, (0, 255, 0), 2)   # 堆積物: 緑
cv2.putText(annotated, label, (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 1.0, (0, 0, 255), 2)
```

**⑧ 保存・記録（第1章）**

```python
cv2.imwrite(ann_path, annotated)
append_log(ts, coverage, clumps)   # CSVに追記（OpenCVではない）
```

これで1回の計測が完了します。あとはこれを cron で繰り返すだけです。

# 第10章 実際に運用する（発展）

> OpenCV の理解は前章で完成しました。最後はおまけの実践編です。
> 学んだプログラムを Raspberry Pi で実際に動かし続けるための運用——キャリブレーション、定期実行、確認画像の取り出し、チューニング——を整理します。
> ここは OpenCV そのものより「使いこなし」の話なので、肩の力を抜いて読んでください。

## 10.1 この章のゴール

この章を終えると、次のことができるようになります。

- 正しい順番で `leaf_monitor.py` を動かし始められる
- `cron` を使って無人で定期計測できる
- 確認画像を PC に取り出して検証できる
- 結果がおかしいとき、どの設定を調整すればよいか見当がつく

---

## 10.2 まず1回キャリブレーション

すべての出発点は、キャリブレーションです。**掃除した直後・堆積物ゼロ・鉢ありの状態**で1回実行します。

```bash
cd <leaf_monitorを置いたパス>   # 例: ~/leaf_monitor （leaf_monitor.py を置いた場所へ）
python3 leaf_monitor.py --calibrate
```

これで、第9章で読んだとおり `mat_config.json`・`region.png`・`pot.png` が作られます。実行後、必ず確認をしてください。

> **確認のしかた**：生成された `pot.png`・`region.png` や、計測を1回回したときの `annotated/<日時>.jpg` を VS Code で開き、**青線＝計測範囲・黄線＝鉢・緑線＝堆積物**が意図どおりかを目で見ます。第8章で学んだ確認画像の読み方が、そのまま運用で役立ちます。

キャリブレーションは「最初に1回」だけでなく、**マット・鉢・照明・カメラの位置を変えたら、そのつどやり直し**が必要です。計測範囲も鉢マスクもピクセル座標で固定保存しているため、どれかが動くと範囲がずれてしまうからです（仕様書 11章）。

## 10.3 単発計測とループ計測

キャリブレーションができたら、計測できます。

```bash
# 1回だけ計測
python3 leaf_monitor.py

# 一定間隔で繰り返し（例：1時間ごと。Ctrl+Cで停止）
python3 leaf_monitor.py --loop 3600
```

計測のたびに、`log.csv` に1行追記され、`annotated/<日時>.jpg` が保存されます。`--loop` は、手元で動作を見ながら試すときに便利です。長期の無人運用には、次の cron のほうが向いています。

---

## 10.4 cron で無人運用する

`cron` は「決まった時刻にコマンドを自動実行する」Linux の仕組みです。WSL2 でも使えるので、考え方は同じです。設定を開きます。

```bash
crontab -e
```

末尾に次のような行を足します（例：毎日 8・12・16 時に計測）。パスは自分の設置場所に合わせてください。

```
0 8,12,16 * * * cd <leaf_monitorを置いた絶対パス> && /usr/bin/python3 leaf_monitor.py >> cron.log 2>&1
```

- `0 8,12,16 * * *` … 「毎日の 8時0分・12時0分・16時0分」という時刻指定。
- `cd ... &&` … スクリプトのある場所へ移動してから実行（出力ファイルの場所を安定させるため）。cron では `~` が使えないことがあるので、`/home/<ユーザー名>/...` のような**絶対パス**で書くのが安全です。
- `>> cron.log 2>&1` … 実行の記録とエラーを `cron.log` に残す。トラブル時の手がかりになります。

これで、PC をつけていなくても Pi が自動で計測を続けてくれます。

---

## 10.5 確認画像を PC に取り出す

annotated 画像は Pi の中にたまっていきます。VS Code の Remote-SSH 経由でそのまま開いてもよいですし、まとめて PC に取り出すなら `scp` が手軽です（Windows の PowerShell や WSL2 のターミナルから）。

```bash
scp <ユーザー名>@<ラズパイのIP>:<leaf_monitorを置いた絶対パス>/annotated/*.jpg .
```

取り出した画像を見て、第8章で学んだ「青・黄・緑」が妥当か、影を誤検出していないかを確認します。検出がずれていたら、次のチューニングに進みます。

---

## 10.6 結果がおかしいときの調整（どの章とつながるか）

仕様書のトラブルシューティング表を、ここまで学んだ知識と結びつけて読み直すと、各設定の意味がはっきり分かります。

| 症状 | 調整 | つながる章 |
|---|---|---|
| マット以外を堆積物と誤検出する | `H/S/V_MARGIN` を広げる、`MIN_MAT_FRAGMENT` を調整 | 第3章（範囲）・第6章（断片の足切り） |
| 堆積物を取りこぼす | 上記マージンを狭める | 第3章 |
| 鉢の縁が堆積物として残る | `POT_DILATE` を大きくして再キャリブレーション | 第4章（膨張） |
| 影を堆積物と誤検出する | 拡散光にする・撮影時刻を固定、`V_MARGIN` を広げる | 第2章（明度）・第3章 |
| マット色が学習されない | マットを大きく写す、`SAT_MIN` などを調整 | 第2章・第3章 |

ポイントは、**「広げると誤検出が増え／狭めると取りこぼす」というトレードオフ**（第3章の「やってみよう」で体験したあれ）が、すべての根っこにあることです。調整したら必ず `--calibrate` をやり直し、確認画像で結果を見る——この繰り返しが運用の基本です。

> **やってみよう**
> わざとマージンを極端に広げて1回計測し、`annotated` 画像で「影や鉢の縁まで緑で拾ってしまう」様子を観察してみましょう。次に狭めすぎて「本物の落ち葉を取りこぼす」様子も見ます。両極端を見ておくと、ちょうどよい設定の勘がつかめます。

---

## 10.7 運用のコツ

仕様書 9.4 にある実践的なコツも、ここで押さえておきましょう。

掃除して堆積物を取り除くと、被覆率の数値はリセット（0付近に戻る）されます。ですから、絶対値そのものより、**清掃サイクル内での「上がり方（増え方の速さ）」**を見るのがトレンド監視に向いています。前日比の差分を取ると、落下が増えた日が分かりやすくなります。

運用を始めたら、まず数日ぶんの「健康なときの平常値（どのくらいの値か、どのくらいの速さで上がるか）」を記録しておきましょう。それを基準に、「この値を超えたら点検・清掃」という目安（しきい値）を決めると、実用的なモニタリングになります。

## 10.8 ここから先（さらに学びたい人へ）

このテキストでは「古典的画像処理」だけを扱いました。これはマットの色を利用した軽量で確実な方法ですが、限界もあります。たとえば、落ち葉と土くず・小枝を**色だけでは区別できません**（仕様書 3.7・11章）。

もし「落ち葉だけを1枚ずつ正確に検出したい」となったら、次のステップは**機械学習（ディープラーニング）**です。仕様書 12章では、YOLO のセグメンテーションモデルへ移行する道筋が示されています。アノテーション（学習用のお手本作り）や計算負荷という新しい苦労が増えますが、重なりや誤検出に強くなります。

ただ、その世界に進むときも、この10章で身につけた土台——「画像は数字の表」「マスク」「面積を数える」「結果を確認画像で見る」——は、そのまま生き続けます。OpenCV の基礎は、どんな画像処理にも共通する出発点だからです。

最後に、全10章で何を学んだかを振り返ります。

- 第0章：ゴールとパイプラインの地図、環境の確認。
