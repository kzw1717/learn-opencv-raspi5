# 第10章 実際に運用する（発展）

> OpenCV の理解は前章で完成しました。最後はおまけの実践編です。
> 学んだプログラムを Raspberry Pi で実際に動かし続けるための運用——キャリブレーション、定期実行、確認画像の取り出し、チューニング——を整理します。
> ここは OpenCV そのものより「使いこなし」の話なので、肩の力を抜いて読んでください。

---

## 10.1 この章のゴール

この章を終えると、次のことができるようになります。

- 正しい順番で `leaf_monitor.py` を動かし始められる
- `cron` を使って無人で定期計測できる
- 確認画像を PC に取り出して検証できる
- 結果がおかしいとき、どの設定を調整すればよいか見当がつく

---

## 10.2 まず1回キャリブレーション

すべての出発点は、キャリブレーションです。**掃除した直後・堆積物ゼロ・鉢ありの状態**で1回実行します。

```bash
cd <leaf_monitorを置いたパス>   # 例: ~/leaf_monitor （leaf_monitor.py を置いた場所へ）
python3 leaf_monitor.py --calibrate
```

これで、第9章で読んだとおり `mat_config.json`・`region.png`・`pot.png` が作られます。実行後、必ず確認をしてください。

> **確認のしかた**：生成された `pot.png`・`region.png` や、計測を1回回したときの `annotated/<日時>.jpg` を VS Code で開き、**青線＝計測範囲・黄線＝鉢・緑線＝堆積物**が意図どおりかを目で見ます。第8章で学んだ確認画像の読み方が、そのまま運用で役立ちます。

キャリブレーションは「最初に1回」だけでなく、**マット・鉢・照明・カメラの位置を変えたら、そのつどやり直し**が必要です。計測範囲も鉢マスクもピクセル座標で固定保存しているため、どれかが動くと範囲がずれてしまうからです（仕様書 11章）。

---

## 10.3 単発計測とループ計測

キャリブレーションができたら、計測できます。

```bash
# 1回だけ計測
python3 leaf_monitor.py

# 一定間隔で繰り返し（例：1時間ごと。Ctrl+Cで停止）
python3 leaf_monitor.py --loop 3600
```

計測のたびに、`log.csv` に1行追記され、`annotated/<日時>.jpg` が保存されます。`--loop` は、手元で動作を見ながら試すときに便利です。長期の無人運用には、次の cron のほうが向いています。

---

## 10.4 cron で無人運用する

`cron` は「決まった時刻にコマンドを自動実行する」Linux の仕組みです。WSL2 でも使えるので、考え方は同じです。設定を開きます。

```bash
crontab -e
```

末尾に次のような行を足します（例：毎日 8・12・16 時に計測）。パスは自分の設置場所に合わせてください。

```
0 8,12,16 * * * cd <leaf_monitorを置いた絶対パス> && /usr/bin/python3 leaf_monitor.py >> cron.log 2>&1
```

- `0 8,12,16 * * *` … 「毎日の 8時0分・12時0分・16時0分」という時刻指定。
- `cd ... &&` … スクリプトのある場所へ移動してから実行（出力ファイルの場所を安定させるため）。cron では `~` が使えないことがあるので、`/home/<ユーザー名>/...` のような**絶対パス**で書くのが安全です。
- `>> cron.log 2>&1` … 実行の記録とエラーを `cron.log` に残す。トラブル時の手がかりになります。

これで、PC をつけていなくても Pi が自動で計測を続けてくれます。

## 10.5 確認画像を PC に取り出す

annotated 画像は Pi の中にたまっていきます。VS Code の Remote-SSH 経由でそのまま開いてもよいですし、まとめて PC に取り出すなら `scp` が手軽です（Windows の PowerShell や WSL2 のターミナルから）。

```bash
scp <ユーザー名>@<ラズパイのIP>:<leaf_monitorを置いた絶対パス>/annotated/*.jpg .
```

取り出した画像を見て、第8章で学んだ「青・黄・緑」が妥当か、影を誤検出していないかを確認します。検出がずれていたら、次のチューニングに進みます。

---

## 10.6 結果がおかしいときの調整（どの章とつながるか）

仕様書のトラブルシューティング表を、ここまで学んだ知識と結びつけて読み直すと、各設定の意味がはっきり分かります。

| 症状 | 調整 | つながる章 |
|---|---|---|
| マット以外を堆積物と誤検出する | `H/S/V_MARGIN` を広げる、`MIN_MAT_FRAGMENT` を調整 | 第3章（範囲）・第6章（断片の足切り） |
| 堆積物を取りこぼす | 上記マージンを狭める | 第3章 |
| 鉢の縁が堆積物として残る | `POT_DILATE` を大きくして再キャリブレーション | 第4章（膨張） |
| 影を堆積物と誤検出する | 拡散光にする・撮影時刻を固定、`V_MARGIN` を広げる | 第2章（明度）・第3章 |
| マット色が学習されない | マットを大きく写す、`SAT_MIN` などを調整 | 第2章・第3章 |

ポイントは、**「広げると誤検出が増え／狭めると取りこぼす」というトレードオフ**（第3章の「やってみよう」で体験したあれ）が、すべての根っこにあることです。調整したら必ず `--calibrate` をやり直し、確認画像で結果を見る——この繰り返しが運用の基本です。

> **やってみよう**
> わざとマージンを極端に広げて1回計測し、`annotated` 画像で「影や鉢の縁まで緑で拾ってしまう」様子を観察してみましょう。次に狭めすぎて「本物の落ち葉を取りこぼす」様子も見ます。両極端を見ておくと、ちょうどよい設定の勘がつかめます。

## 10.7 運用のコツ

仕様書 9.4 にある実践的なコツも、ここで押さえておきましょう。

掃除して堆積物を取り除くと、被覆率の数値はリセット（0付近に戻る）されます。ですから、絶対値そのものより、**清掃サイクル内での「上がり方（増え方の速さ）」**を見るのがトレンド監視に向いています。前日比の差分を取ると、落下が増えた日が分かりやすくなります。

運用を始めたら、まず数日ぶんの「健康なときの平常値（どのくらいの値か、どのくらいの速さで上がるか）」を記録しておきましょう。それを基準に、「この値を超えたら点検・清掃」という目安（しきい値）を決めると、実用的なモニタリングになります。

## 10.8 ここから先（さらに学びたい人へ）

このテキストでは「古典的画像処理」だけを扱いました。これはマットの色を利用した軽量で確実な方法ですが、限界もあります。たとえば、落ち葉と土くず・小枝を**色だけでは区別できません**（仕様書 3.7・11章）。

もし「落ち葉だけを1枚ずつ正確に検出したい」となったら、次のステップは**機械学習（ディープラーニング）**です。仕様書 12章では、YOLO のセグメンテーションモデルへ移行する道筋が示されています。アノテーション（学習用のお手本作り）や計算負荷という新しい苦労が増えますが、重なりや誤検出に強くなります。

ただ、その世界に進むときも、この10章で身につけた土台——「画像は数字の表」「マスク」「面積を数える」「結果を確認画像で見る」——は、そのまま生き続けます。OpenCV の基礎は、どんな画像処理にも共通する出発点だからです。

## 10.9 このテキスト全体のまとめ

最後に、全10章で何を学んだかを振り返ります。

- 第0章：ゴールとパイプラインの地図、環境の確認。
- 第1章：画像は数字の表（NumPy 配列）。BGR の順番。`imread`/`imwrite`。
- 第2章：HSV で色味と明るさを分ける。`cvtColor`。
- 第3章：色の範囲を白く抜き出すマスク。`inRange` とマージン。
- 第4章：マスクの掃除。`morphologyEx`（OPEN/CLOSE）・`dilate`。
- 第5章：マスクの組み合わせ。`bitwise_not`・`bitwise_and`で堆積物を取り出す。
- 第6章：輪郭・凸包・塗りつぶし。`findContours`・`contourArea`・`convexHull`・`fillPoly`。
- 第7章：白い画素を数えて被覆面積率。`countNonZero`と「分母の固定」。
- 第8章：確認画像を描く。`drawContours`・`putText`、再びBGR。
- 第9章：`leaf_monitor.py` を通読し、すべての道具を回収。
- 第10章：実際の運用とチューニング、その先の発展。

「マットの上の落ち葉が何パーセントを覆っているか」——一見すると難しそうなこの計測が、素朴な道具を一段ずつ積み重ねるだけで実現できる。これが、画像処理のおもしろさであり、このテキストで伝えたかったことです。
