# 第7章 面積を測る（被覆面積率）

> いよいよ、このプログラムの主役の数字「**被覆面積率（％）**」を計算します。
> 仕組みは拍子抜けするほど素朴で、「白い画素を数えて、割り算する」だけ。
> でもその裏には「分母を固定する」という、トレンド監視のための大事な工夫が隠れています。

---

## 7.1 この章のゴール

この章を終えると、次のことができるようになります。

- `cv2.countNonZero` で、マスクの白い画素の数（＝面積）を数えられる
- 「被覆面積率＝堆積物の面積 ÷ 計測範囲の面積 × 100」を計算できる
- なぜ計測範囲（分母）を**固定**しなければならないかを説明できる
- マスクをファイルに保存し、`IMREAD_GRAYSCALE` で読み戻す流れを理解できる

---

## 7.2 面積を数える：countNonZero

マスクは白（255）と黒（0）だけの画像でした。白い画素1つを「1ピクセルの面積」と考えれば、**白い画素の数を数えれば、それがそのまま面積**になります。

`cv2.countNonZero` は、その名のとおり「0でない画素（＝白い画素）の数」を数えてくれる関数です。

```python
# step20.py
import cv2

region = cv2.imread("mat_region.jpg", cv2.IMREAD_GRAYSCALE)   # 第6章で作ったマット外形
leaf   = cv2.imread("leaf.jpg",       cv2.IMREAD_GRAYSCALE)   # 第5章で作った堆積物

region_area = cv2.countNonZero(region)
leaf_area   = cv2.countNonZero(leaf)

print("計測範囲の面積:", region_area, "px")
print("堆積物の面積  :", leaf_area,   "px")
```

```bash
python3 step20.py
```

それぞれの白い画素数（ピクセル単位の面積）が表示されます。これで「全体の広さ」と「覆われている広さ」の両方が手に入りました。

---

## 7.3 割り算して「被覆面積率」を出す

あとは割り算です。「覆われている面積」を「全体の面積」で割り、100 を掛ければパーセントになります。

```python
# step21.py
import cv2

region = cv2.imread("mat_region.jpg", cv2.IMREAD_GRAYSCALE)
leaf   = cv2.imread("leaf.jpg",       cv2.IMREAD_GRAYSCALE)

region_area = cv2.countNonZero(region)
leaf_area   = cv2.countNonZero(leaf)

coverage_pct = 100.0 * leaf_area / region_area if region_area else 0.0
print(f"被覆面積率 = {coverage_pct:.2f}%")
```

`if region_area else 0.0` は、「範囲の面積が0のときに0で割ってエラーになるのを防ぐ」ための安全装置です（割り算の前に0でないか確認している）。

これで、`leaf_monitor.py` が CSV に記録している主役の数字「`coverage_pct`」が、自分の手で計算できました。仕組みは「**白い画素を数えて割る**」だけ。とても素朴ですね。

> **やってみよう 1**
> マットの上に落ち葉を数枚足して、もう1枚撮影し（`rpicam-jpeg`）、第3章〜第5章と同じ手順で堆積物マスクを作り直してから、もう一度この計算をしてみましょう。落ち葉が増えたぶん、`coverage_pct` が**上がる**はずです。これがまさに「増加トレンドを追う」ということです。

---

## 7.4 いちばん大事な工夫：分母を固定する

ここで、`leaf_monitor.py` の設計の核心に触れます。仕様書「3.6 計測範囲：キャリブレーション時に固定」で説明されている、重要な考え方です。

被覆面積率は「堆積物の面積 ÷ **計測範囲の面積**」でした。この**分母（計測範囲の面積）が日によって変わってしまうと、値を正しく比べられません**。

なぜ変わりうるのでしょうか。もし毎回その日の写真からマット外形を計算し直すと、堆積物が増えてマットが隠れていくにつれ、見えるマットがどんどん小さくなります。すると分母も小さくなり、「分母も分子も動く」状態になって、トレンドが崩れてしまいます。

そこで `leaf_monitor.py` は、**マットがきれいな最初の状態で計測範囲を1回だけ確定し、ファイル（`region.png`）に保存して固定**します。以降の計測では、毎回この同じ範囲を分母に使います。

> 分母が毎回同じ ＝ 「ものさし」が変わらない。だから日々の被覆率の増減が、そのまま堆積物の増減を表す。

これが、トレンド監視を正しく行うためのいちばんの肝です。私たちが `step20.py`〜`step21.py` で `mat_region.jpg` を毎回読み込んで使ったのも、同じ「決まったものさしを使い回す」発想です。

---

## 7.5 マスクを保存して、読み戻す

「固定する」を実現するには、計測範囲のマスクをファイルに保存しておき、計測のたびに読み戻す必要があります。第1章で学んだ `cv2.imwrite` と `cv2.imread` の出番です。

- 保存：`cv2.imwrite("region.png", region)` … キャリブレーション時に1回だけ。
- 読み戻し：`cv2.imread("region.png", cv2.IMREAD_GRAYSCALE)` … 計測のたびに。

ここで `cv2.IMREAD_GRAYSCALE` を付けているのがポイントです。マスクは白黒なので、カラー（3つ組）ではなく**白黒（1マス1個）で読み込む**よう指定しています。こうしておけば、`countNonZero` などの白黒マスク向けの処理にそのまま渡せます。

> **やってみよう 2**
> `cv2.imwrite("region.png", region)` で保存し、別のスクリプトから `cv2.imread("region.png", cv2.IMREAD_GRAYSCALE)` で読み戻して、`countNonZero` の値が保存前と同じになることを確認しましょう。「一度決めた範囲を、別の実行でも同じものさしとして使える」ことが体感できます。

## 7.6 `leaf_monitor.py` への橋渡し

この章の計算は、`analyze()` の中心部にそのまま現れます。

```python
region_area = int(cv2.countNonZero(region))   # ★ 固定なので毎回同じ
leaf_area   = int(cv2.countNonZero(leaf_mask))
coverage_pct = 100.0 * leaf_area / region_area if region_area else 0.0
```

`step20.py`〜`step21.py` とまったく同じです。コメントの「★ 固定なので毎回同じ」が、7.4 で学んだ「分母を固定する」工夫を指しています。

そして、その「固定の範囲」は、計測のたびに `run_once()` の中でファイルから読み戻されています。

```python
region = cv2.imread(REGION_MASK, cv2.IMREAD_GRAYSCALE)
```

`cv2.IMREAD_GRAYSCALE` で白黒のまま読み込んでいるのも、私たちがやったとおりです。`REGION_MASK` は `region.png` のことで、これはキャリブレーション時に `calibrate()` の中で `cv2.imwrite(REGION_MASK, region)` として一度だけ保存されたものです。

「最初に1回だけものさしを作って保存 → あとはそれを読み戻して使い続ける」。この流れが、トレンドを正しく追うための土台になっています。

## 7.7 この章のまとめ

- `cv2.countNonZero(mask)` は、マスクの白い画素の数＝面積（ピクセル）を数える。
- 被覆面積率 ＝ `100 × 堆積物の面積 ÷ 計測範囲の面積`。0で割らないよう安全装置を入れる。
- 分母（計測範囲）を毎回作り直すとトレンドが崩れる。だから**最初に1回固定**し、`region.png` に保存して使い回す。
- 固定マスクは `cv2.imwrite` で保存し、計測のたびに `cv2.imread(..., cv2.IMREAD_GRAYSCALE)` で読み戻す。
