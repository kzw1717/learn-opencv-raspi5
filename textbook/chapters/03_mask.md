# 第3章 マスクを作る（色の抜き出し）

> HSV にできたので、いよいよ「マットの色だけ」を抜き出します。
> 抜き出した結果は、白と黒だけの画像 ＝ **マスク（型紙）** になります。
> この章の `cv2.inRange` は、`leaf_monitor.py` がマットを認識する、まさにその瞬間の道具です。

---

## 3.1 この章のゴール

この章を終えると、次のことができるようになります。

- **マスク**とは「白＝対象、黒＝対象外」の白黒画像（型紙）だと説明できる
- `cv2.inRange` を使って、指定した HSV の範囲に入る画素だけを白く抜き出せる
- しきい値（下限・上限）を変えると、抜き出される範囲がどう変わるかを体験できる
- `leaf_monitor.py` の `H_MARGIN` / `S_MARGIN` / `V_MARGIN` が何を決めているか理解できる

---

## 3.2 マスクってなに？

**マスク**は、もとの画像と同じ大きさの白黒画像です。

- 白（255）＝「ここが対象だよ」という印
- 黒（0）＝「ここは対象じゃないよ」という印

工作で使う「型紙」を思い浮かべてください。型紙の穴が開いているところ（白）だけ絵の具が通り、それ以外（黒）は通さない——あの感覚です。画像処理では、このマスクを使って「どこを処理するか／しないか」を指定します。

`leaf_monitor.py` の中でも、`region.png`（計測範囲）や `pot.png`（鉢）など、いくつものマスクが活躍します。マスクは画像処理の主役級の道具なので、ここでしっかりつかんでおきましょう。

---

## 3.3 cv2.inRange で範囲内を白くする

`cv2.inRange` は、「指定した範囲に入っている画素を白、入っていない画素を黒」にした**マスクを作る**関数です。

```python
mask = cv2.inRange(hsv, 下限, 上限)
```

`下限`・`上限` は、HSV それぞれの「ここからここまで」を表す3つ組です。たとえば「H が 100〜140、S が 80〜255、V が 50〜255 の画素を白くする」なら、下限が `[100, 80, 50]`、上限が `[140, 255, 255]` です。

第2章で撮った `mat.jpg` を使って、青いマットを抜き出してみましょう。

```python
# step8.py
import cv2
import numpy as np

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# 青あたり（H=120付近）を抜き出す。値はマットの色に合わせて後で調整する
lower = np.array([100, 80, 50])
upper = np.array([140, 255, 255])

mask = cv2.inRange(hsv, lower, upper)

print("マスクの形:", mask.shape)
print("マスクに含まれる値:", np.unique(mask))   # 0と255だけのはず

cv2.imwrite("mat_mask.jpg", mask)
print("mat_mask.jpg を保存しました。")
```

```bash
python3 step8.py
```

出力の例です。

```
マスクの形: (1520, 2028)
マスクに含まれる値: [  0 255]
```

ここで2つ大事なことがあります。第一に、マスクの形は `(縦, 横)` で、**3つ組ではなく1マス1個の数字**です（白黒だから色は要りません）。第二に、中身は **0 と 255 しかありません**。まさに白と黒だけの型紙です。

`mat_mask.jpg` を VS Code で開いてみてください。マットの部分が**白く**、それ以外（鉢・落ち葉・背景）が**黒く**なっていれば成功です。

> もし白い部分が少なすぎる／多すぎる場合は、`lower` と `upper` の数字がマットの色に合っていません。次の節で調整します。

## 3.4 しきい値を変えて遊ぶ（いちばん大事な実験）

この章の本番はここです。**範囲を変えると抜き出され方がどう変わるか**を、手で動かして体に入れます。

まず、自分のマットの実際の HSV を調べましょう。`mat.jpg` をVS Codeで開き、マットの中央あたりがだいたい画像のどの位置か見当をつけて、その HSV を表示します。

```python
# step9.py
import cv2

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# マットの中央あたり（座標は自分の写真に合わせて変える）
print("マット中央の HSV:", hsv[760, 1000])
```

たとえば `[118 170 190]` と出たら、マットの色相 H は 118 あたりだと分かります。この値を中心に、`step8.py` の範囲を作り直してみましょう。「中心 ± 余裕（マージン）」で範囲を作るのがコツです。

```python
# step10.py
import cv2
import numpy as np

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

center_h = 118        # ← step9.py で調べた値に置き換える

# マージン（許容幅）。ここを変えて結果の変化を見るのが今回の実験
H_MARGIN = 12
S_MARGIN = 70
V_MARGIN = 80

s, v = 170, 190       # マット中央の S, V（step9で調べた値）
lower = np.array([center_h - H_MARGIN, max(0, s - S_MARGIN), max(0, v - V_MARGIN)])
upper = np.array([center_h + H_MARGIN, min(255, s + S_MARGIN), min(255, v + V_MARGIN)])

mask = cv2.inRange(hsv, lower, upper)
cv2.imwrite("mat_mask2.jpg", mask)
print("下限:", lower, " 上限:", upper)
print("mat_mask2.jpg を保存しました。")
```

> **やってみよう 1**
> `H_MARGIN` を `12 → 30` に増やして実行し、`mat_mask2.jpg` がどう変わるか見てください。白い部分が**広がる**はずです。範囲をゆるくすると、マットに近い色のものまで「マット」と判定されるようになります。

> **やってみよう 2**
> 逆に `H_MARGIN` を `12 → 4` に減らすと、白い部分が**狭く**なり、マットの一部が黒く欠けることもあります。範囲を厳しくしすぎると、本来のマットまで取りこぼします。

この「広げすぎると余計なものまで拾う／狭めすぎると取りこぼす」というトレードオフが、まさに `leaf_monitor.py` のチューニングの勘どころです。仕様書のトラブルシューティング表にある「誤検出が多ければマージンを広げ、取りこぼしが多ければ狭める」は、この実験そのものなのです。

> **やってみよう 3**
> `S_MARGIN`・`V_MARGIN` も変えてみましょう。S を厳しくすると、白っぽい・くすんだ部分が外れやすくなります。V を広げると、影になったマットも拾いやすくなります。`leaf_monitor.py` で V_MARGIN が広め（80）に設定されているのはなぜか、考えてみましょう。

## 3.5 赤い色のちょっとした特殊事情（軽く知っておく）

ここは「へぇ」と思う程度で大丈夫です。第2章で「H は 0〜180 の円」と学びました。**赤は、その円のつなぎ目（0 と 180 の両方）にまたがっています**。そのため「赤を抜き出したい」ときだけは、`cv2.inRange` を1回では指定できず、0付近の範囲と180付近の範囲を**2回に分けて**抜き出し、あとで合体させる必要があります。

`leaf_monitor.py` の `get_mat_mask()` は、この赤の特殊事情にも対応できるよう作られています。ただし**青や緑のマットを使う分には、ふつうの1回の `inRange` で済みます**。今回は深入りせず、「赤マットを使うと少しややこしい」とだけ覚えておけば十分です（だから仕様書でも青・緑が推奨されています）。

## 3.6 `leaf_monitor.py` への橋渡し

この章の主役 `cv2.inRange` は、`leaf_monitor.py` の `get_mat_mask()` の中心です。

```python
return cv2.inRange(hsv, lower.astype(np.uint8), upper.astype(np.uint8))
```

私たちが `step8.py` でやったのと同じく、HSV 画像から「マット色の範囲」を白く抜き出しています。

そして `lower` と `upper` がどう決まるかというと、`calibrate()` の中で「マットの代表色 ± マージン」で計算されています。

```python
lower = [peak - H_MARGIN, max(0, s_med - S_MARGIN), max(0, v_med - V_MARGIN)]
upper = [peak + H_MARGIN, min(255, s_med + S_MARGIN), min(255, v_med + V_MARGIN)]
```

`step10.py` で私たちが手作業で組み立てた式と、まったく同じ形をしています。`peak`（マットの代表的な色相）は、第2章でふれた「色相ヒストグラムの最頻値」、つまり**一番面積を占める色＝マットの色**です。`H_MARGIN`・`S_MARGIN`・`V_MARGIN` は、いま「やってみよう」で動かしてみた、あの許容幅そのものです。

> なお `leaf_monitor.py` では、`inRange` の前に S や V の最低・最高ライン（`SAT_MIN`・`VAL_MIN`・`VAL_MAX`）も使って、灰色・黒・白飛びした画素を「色なし」として代表色の計算から外しています。これも「マットの色味だけを正しくつかむ」ための工夫です。

---

## 3.7 この章のまとめ

- **マスク**は「白＝対象・黒＝対象外」の白黒画像（型紙）。画像処理の主役級の道具。
- `cv2.inRange(hsv, 下限, 上限)` で、範囲内の画素を白くしたマスクを作れる。中身は 0 と 255 だけ。
- 範囲を**広げると余計なものまで拾い、狭めると取りこぼす**。このトレードオフがチューニングの勘どころ。
- `H_MARGIN`・`S_MARGIN`・`V_MARGIN` は、この範囲の許容幅を決める設定。
- 赤はH円のつなぎ目にまたがるので少し特殊。青・緑マットなら気にしなくてよい。

## 3.8 確認問題

1. `cv2.inRange` が返すマスクには、どんな値だけが含まれるか答えなさい。また、その `shape` がもとのカラー画像と比べてどう違うか説明しなさい。
2. マットの取りこぼし（マットなのに黒くなる部分）が多いとき、マージンを広げるべきか狭めるべきか答えなさい。
3. マット以外のものまで白く拾ってしまう（誤検出が多い）とき、マージンをどうすべきか答えなさい。
4. `leaf_monitor.py` の `lower`・`upper` の式と、`step10.py` であなたが書いた式は、本質的に同じことをしている。何を計算しているのか、共通点を説明しなさい。
