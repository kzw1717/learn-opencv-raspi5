## 0.1 私たちがたどり着く場所

このテキストの最終ゴールは、`leaf_monitor.py` という1本のプログラムに出てくる **OpenCV の機能を、すべて自分の言葉で説明できる**ようになることです。

`leaf_monitor.py` は、Raspberry Pi のカメラで鉢の周りを定期的に撮影し、**色付きのマットの上に積もった落ち葉やゴミ（堆積物）が、マットの何パーセントを覆っているか**を自動で計算して記録するプログラムです。この「覆っている割合」を時間とともに記録していくことで、ミニトマトの落葉の傾向＝健康状態のトレンドを見守ることができます。

このプログラムが最終的に作り出すのが、次のような**確認用の画像**です（言葉でイメージしてください）。

- もとの写真の上に、**青い線**で「計測する範囲」が描かれている
- **黄色い線**で「除外する鉢」が囲まれている
- **緑の線**で「見つけた堆積物（落ち葉など）」が囲まれている
- 左上に `coverage=12.34%  clumps=5` のように、覆われた割合と塊の数が書かれている

私たちの目標は、この1枚の画像が**どうやって作られているのか**を、最初から最後まで理解することです。難しそうに見えますが、分解すれば一つひとつは素朴な作業の積み重ねです。

## 0.2 全体の流れ（このテキストの地図）

`leaf_monitor.py` がやっている画像処理は、次のような一本道のパイプラインです。各ステップが、このテキストの各章に対応しています。

```
写真を読み込む                          … 第1章（画像＝数字の表）
    ↓
色を「色味」で扱える形にする（HSV）       … 第2章
    ↓
マットの色だけを白く抜き出す（マスク）    … 第3章
    ↓
マスクのノイズや穴をきれいにする          … 第4章
    ↓
マスク同士を組み合わせて堆積物を取り出す  … 第5章
    ↓
形（輪郭）を見つけ、範囲を確定する        … 第6章
    ↓
白い画素を数えて「覆われた割合」を出す    … 第7章
    ↓
結果を画像に描いて確認できるようにする    … 第8章
    ↓
本物の leaf_monitor.py を通して読む       … 第9章
    ↓
実際に Pi で運用する（発展）             … 第10章
```

各章の終わりには、必ず「いま学んだことが `leaf_monitor.py` のどの行に効いているか」を確認します。最終章にたどり着くころには、本物のコードを上から読みながら「これは○章でやったやつだ」とすべて回収できる状態になっています。

## 0.3 環境の確認

このテキストは、次の環境を前提にしています。

- 本体：Raspberry Pi 5（カメラ：Camera Module v1.3 / OV5647）
- OS：Raspberry Pi OS Lite（画面のない、コマンドライン中心の環境）
- 開発：Windows 11 の VS Code から **Remote-SSH** で Pi につなぎ、Pi 上で直接ファイルを編集・実行する

普段 WSL2 の Ubuntu で Python を書いているなら、操作感はほとんど同じです。違うのは「動いている場所が手元の PC ではなく Pi の上」という点だけです。

まず、必要なライブラリが入っているかを確認します。VS Code のターミナル（Pi 側のシェル）で次を実行してください。

```bash
python3 -c "import cv2, numpy; print('OpenCV', cv2.__version__, '/ NumPy', numpy.__version__)"
```

`OpenCV 4.x.x / NumPy 1.x.x` のようにバージョンが表示されれば準備 OK です。もし `ModuleNotFoundError` が出たら、ライブラリが未インストールなのでインストールします。

```bash
sudo apt update
sudo apt install -y python3-opencv python3-numpy
```

カメラが認識されているかも見ておきましょう。エラーが出なければ大丈夫です。

```bash
rpicam-hello --timeout 2000
```

> **補足：なぜ `pip install opencv-python` ではないのか**
> Pi では `apt` で入る `python3-opencv` を使うのが最も確実です。`pip` 版は Pi 向けのビルドで不具合が出ることがあり、カメラ用ライブラリ picamera2 とも相性問題が起きやすいためです。WSL2 では `pip` で入れることもありますが、Pi では `apt` 優先、と覚えておいてください。

## 0.5 「見える化」の方法を先に決めておく

画像処理の勉強でいちばん大事なのは、**処理の前後を必ず目で見て確かめる**ことです。文字だけ追っていると、マスクやノイズ除去が何をしているのかまったく分かりません。

ところが私たちの Raspberry Pi OS Lite には画面（デスクトップ）がありません。そのため、よく入門書に出てくる `cv2.imshow()`（ウィンドウを開いて画像を表示する関数）は**使えません**。

そこでこのテキストでは、最初から最後まで次のやり方で確認します。

> **画像をファイルに保存する（`cv2.imwrite`）→ VS Code の左側のファイル一覧でクリックして開く**

VS Code は Remote-SSH 経由でも画像ファイルをそのまま表示できます。地味ですが、この環境ではこれが一番確実で速い方法です。覚えておいてください。

## 0.6 NumPy の最低限のおさらい

OpenCV の画像は、その正体が **NumPy の配列**です。WSL2 で少し触ったことがあるはずですが、このテキストで出てくる範囲だけ、軽くおさらいしておきます。詳しくは各章で必要になったときに改めて説明します。

| 書き方 | 意味 |
|---|---|
| `arr.shape` | 配列の形（大きさ）。画像なら `(縦, 横, 色数)` |
| `arr.dtype` | 1マスに入る数字の種類。画像は `uint8`（0〜255の整数） |
| `arr[100, 200]` | 上から100、左から200の位置の値を取り出す |
| `arr[:, :, 0]` | 3つ組のうち0番目だけを取り出す（スライス） |
| `np.zeros(形, dtype=np.uint8)` | すべて0（真っ黒）の配列を作る |
| `np.array([...])` | リストから配列を作る |

これくらいが分かっていれば、各章を読み進められます。「画像＝NumPy 配列」という感覚だけ、強く持っておいてください。

## 0.7 この章のまとめ

- 最終ゴールは、`leaf_monitor.py` の OpenCV をすべて自分の言葉で説明できるようになること。
- そのプログラムは「マットがどれだけ堆積物に覆われたか」を割合で計算し、確認用の画像（青＝範囲・黄＝鉢・緑＝堆積物）を作る。
- 学習は「写真→HSV→マスク→掃除→組み合わせ→輪郭→面積→描画」という一本道で進む。
- 画面のない Pi では `cv2.imshow` は使えないので、**保存して VS Code で開く**のが共通の確認方法。
- 画像の正体は NumPy 配列。最低限の操作をおさらいしておいた。

準備ができたら、いよいよ第1章で「画像は数字の表である」という、いちばん大事な発想転換から始めます。
