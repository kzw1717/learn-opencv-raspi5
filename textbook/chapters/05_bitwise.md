# 第5章 マスクを組み合わせる（ビット演算）

> ここまでで、きれいなマスクが作れるようになりました。
> この章では複数のマスクを「かつ」「でない」で組み合わせて、**「計測範囲の中で・マット色でないもの＝堆積物」** を取り出します。
> `leaf_monitor.py` が落ち葉を抜き出す、いちばん核心の1行を読み解く章です。

---

## 5.1 この章のゴール

この章を終えると、次のことができるようになります。

- マスク同士を「論理」で組み合わせるという発想を理解できる
- `cv2.bitwise_not`（白黒を反転＝「でない」）が使える
- `cv2.bitwise_and`（両方が白の所だけ残す＝「かつ」）が使える
- 「範囲の中で・マット色でない部分＝堆積物」を、自分で組み立てられる
- `leaf_monitor.py` の落ち葉抽出の核心を読み解ける

---

## 5.2 マスクは「論理」で組み合わせられる

マスクは白（255）と黒（0）だけの画像でした。白を「はい（真）」、黒を「いいえ（偽）」と考えると、マスク同士を**論理**で組み合わせられます。

- **NOT（でない）**：白と黒をひっくり返す。「マット色」→「マット色でない」。
- **AND（かつ）**：両方が白の場所だけ白く残す。「範囲の中」かつ「マット色でない」。
- （OR（または）：どちらかが白なら白。今回の主役ではありませんが、赤マットの2範囲の合体などで使われます。）

私たちがやりたいのは、まさにこれです。

> **堆積物 ＝ 「計測範囲の中」 かつ 「マット色でない」**

この日本語の文を、そのまま `AND` と `NOT` で組み立てれば、堆積物のマスクができあがります。

---

## 5.3 NOT：マスクを反転する

まず「マット色でない」を作ります。第3章で作った「マット色のマスク」を反転すればよいですね。`cv2.bitwise_not` を使います。

```python
# step14.py
import cv2

mat_mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)   # マット色＝白

not_mat = cv2.bitwise_not(mat_mask)   # 白黒を反転 ＝ マット色でない＝白

cv2.imwrite("not_mat.jpg", not_mat)
print("not_mat.jpg を保存しました。白黒が反転したか確認してください。")
```

`mat_mask.jpg` と `not_mat.jpg` を見比べてください。**白と黒が完全に入れ替わって**います。`not_mat.jpg` では、マット以外（鉢・落ち葉・背景）が白くなっているはずです。

ただし、この `not_mat.jpg` には鉢も背景も全部入ってしまっています。私たちが欲しいのは「**計測範囲の中の**マット色でない部分」だけ。そこで次の AND で範囲を絞ります。

---

## 5.4 AND：2つのマスクの「重なり」だけ残す

`cv2.bitwise_and` は、2つのマスクの**両方が白の場所だけ**を白く残します。「重なり（共通部分）」を取り出すイメージです。

「計測範囲」のマスクが必要ですが、これは本来は第6章でちゃんと作ります。この章では練習として、画像の中央の長方形を「仮の計測範囲」として手作りして、仕組みを体験しましょう。

```python
# step15.py
import cv2
import numpy as np

mat_mask = cv2.imread("mat_mask.jpg", cv2.IMREAD_GRAYSCALE)
h, w = mat_mask.shape

# 仮の計測範囲：中央の大きな長方形を白く塗ったマスク（本番は第6章で作る）
region = np.zeros((h, w), dtype=np.uint8)
cv2.rectangle(region, (w//5, h//5), (w*4//5, h*4//5), 255, -1)  # -1で内部を塗りつぶし

# 「マット色でない」
not_mat = cv2.bitwise_not(mat_mask)

# 「範囲の中」かつ「マット色でない」＝堆積物
leaf = cv2.bitwise_and(region, not_mat)

cv2.imwrite("region.jpg", region)
cv2.imwrite("leaf.jpg", leaf)
print("region.jpg（仮の範囲）と leaf.jpg（取り出した堆積物）を保存しました。")
```

`region.jpg`・`leaf.jpg` を VS Code で開いてください。

- `region.jpg` は、中央の長方形だけが白い「仮の計測範囲」です。
- `leaf.jpg` は、その**長方形の中にある・マット色でないもの**だけが白く残っています。範囲の外（長方形の外）は、たとえマット色でなくても黒くなっています。

これがまさに「**範囲の中** かつ **マット色でない** ＝ 堆積物」です。日本語の文が、そのまま `bitwise_and(region, bitwise_not(mat_mask))` という式になりました。

> **やってみよう 1**
> `step15.py` の `region` の長方形を小さくしたり大きくしたりして、`leaf.jpg` の白い範囲がそれに合わせて変わることを確認しましょう。AND が「両方が白の場所だけ」を残していることが体感できます。

> **やってみよう 2**
> `cv2.bitwise_and(region, not_mat)` の代わりに `cv2.bitwise_and(region, mat_mask)` にすると、何が取り出されるでしょうか? 予想してから実行して確かめてみてください（「範囲の中のマット部分」が出るはずです）。

## 5.5 同じ考え方で「鉢」も取り出せる

この AND と NOT の組み合わせは、`leaf_monitor.py` のキャリブレーション（最初の準備）でも大活躍します。

落ち葉がない状態で撮った写真では、「マットの範囲の中で・マット色でないもの」は何でしょうか? 落ち葉がないのですから、それは**鉢などの固定物**です。つまり、

> 鉢 ＝ 「マット範囲の中」 かつ 「マット色でない」

これも同じ式で書けます。落ち葉がある計測時には「マット色でないもの＝堆積物」、落ち葉がないキャリブレーション時には「マット色でないもの＝鉢」。同じ仕組みを、状況によって違う目的に使い回しているわけです。とても賢い設計です。

---

## 5.6 `leaf_monitor.py` への橋渡し

この章の核心は、`analyze()` の中のこの1行に凝縮されています。

```python
leaf_mask = cv2.bitwise_and(region, cv2.bitwise_not(mat_color))
```

`step15.py` で私たちが書いた `cv2.bitwise_and(region, not_mat)` と、**まったく同じ形**です。

- `region` … 固定された計測範囲のマスク（第6章・第7章で詳しく扱います）
- `cv2.bitwise_not(mat_color)` … 「マット色でない」（私たちの `not_mat`）
- `bitwise_and(...)` … 「範囲の中」かつ「マット色でない」＝堆積物

この1行で「堆積物の場所」が確定します。`leaf_monitor.py` のいちばん肝心な部分が、第3章のマスク・第4章の掃除・この章のビット演算という、ここまで学んだ3つの組み合わせでできていることが分かります。

そしてキャリブレーション側では、`calibrate()` に次の行があります。

```python
pot = cv2.bitwise_and(mat_region, cv2.bitwise_not(mat_color))
```

5.5 で話した「鉢 ＝ マット範囲 かつ マット色でない」が、この1行です。同じ道具で鉢を取り出しているのが見て取れます。

---

## 5.7 この章のまとめ

- マスクの白を「真」、黒を「偽」と考えると、マスク同士を**論理**で組み合わせられる。
- `cv2.bitwise_not(mask)` … 白黒を反転（「〜でない」）。「マット色」→「マット色でない」。
- `cv2.bitwise_and(a, b)` … 両方が白の場所だけ残す（「かつ」）。範囲の絞り込みに使う。
- 「**範囲の中** かつ **マット色でない**＝堆積物」が、そのまま `bitwise_and(region, bitwise_not(mat))` になる。
- 同じ式が、計測時は「堆積物」、キャリブレーション時は「鉢」を取り出すのに使い回される。
