# 第2章 色を分けて考える（HSV）

> 第1章で、画像は「BGR（青・緑・赤）の数字の表」だと分かりました。
> でも実は、この BGR のままでは「マットの青色だけを取り出す」のがとても難しいのです。
> この章では、色を扱いやすくする魔法の変換 **HSV** を学びます。`leaf_monitor.py` がマットの色を見分けられる、その心臓部です。

## 2.1 この章のゴール

この章を終えると、次のことができるようになります。

- BGR のままだと色の指定が難しい理由を説明できる
- **HSV**（色相・彩度・明度）が、色を「色味」と「明るさ」に分けて扱える仕組みだと理解できる
- `cv2.cvtColor` で画像を HSV に変換し、H・S・V それぞれを画像として確認できる
- 「照明が変わっても色味（H）はあまり動かない」ことを、自分の目で確かめる

---

## 2.2 なぜ BGR だと困るのか

たとえば「青いマットの部分だけを取り出したい」とします。BGR で考えると、青は `B` が大きい色です。では「`B` が 150 以上なら青」と決めれば良さそうに見えます。

ところが現実はそう簡単ではありません。同じ青いマットでも、

- 日なたの明るい部分：`B` がとても大きい（たとえば 230）
- 日かげの暗い部分：`B` が小さい（たとえば 90）

このように、**同じ色なのに明るさで数字が大きく変わってしまう**のです。BGR では「色味」と「明るさ」がぐちゃっと混ざっているため、「明るくても暗くても青いものだけ」をうまく言い表せません。

自然光の下で1日中撮影する `leaf_monitor.py` では、これは致命的です。朝・昼・夕方で明るさが変わるたびに、マットの判定がブレてしまいます。

---

## 2.3 HSV という考え方

そこで登場するのが **HSV** です。HSV は色を、次の3つの要素に分けて表します。

- **H（Hue・色相）**：色味そのもの。「赤っぽい・青っぽい・緑っぽい」という色の種類。
- **S（Saturation・彩度）**：鮮やかさ。大きいほど鮮やか、小さいほど白っぽい・灰色っぽい。
- **V（Value・明度）**：明るさ。大きいほど明るい、小さいほど暗い。

ここが肝心です。HSV では、**明るさは V だけが担当**します。だから、

> 日なたでも日かげでも、変わるのは主に V（明るさ）だけ。色味の H はだいたい同じ。

となります。「明るさが変わっても、色味で物を見分ける」——これがまさに `leaf_monitor.py` の戦略です。

> **OpenCV ならではの注意：H の範囲は 0〜180**
> 色相は本来 0〜360 度の円で表しますが、OpenCV では 1マスに収まる 0〜255 の数字で表すため、半分にして **0〜180** にしています。「赤は 0 付近」「緑は 60 付近」「青は 120 付近」が目安です。一方、S と V は 0〜255 です。範囲が H だけ違う点に注意してください。

## 2.4 cvtColor で HSV に変換する

第1章で白黒に変換するのに使った `cv2.cvtColor`。同じ関数で、今度は HSV に変換します。変換先を `cv2.COLOR_BGR2HSV` に変えるだけです。

このあとの章でずっと使う、マット＋落ち葉のサンプル写真を撮っておきましょう。色付きマット（青か緑が推奨）の上に、落ち葉や小さなゴミをいくつか置いた状態で撮ります。

```bash
cd ~/opencv_lesson
rpicam-jpeg -o mat.jpg --timeout 2000
```

撮れたら `mat.jpg` を VS Code で開いて、マットと落ち葉が写っていることを確認してください。では変換してみます。

```python
# step5.py
import cv2

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

print("もとの画像の形:", img.shape)
print("HSVの画像の形 :", hsv.shape)

# HSVのある1点をのぞいてみる
print("ある画素のHSV:", hsv[100, 200])
```

```bash
python3 step5.py
```

出力はたとえばこうなります。

```
もとの画像の形: (1520, 2028, 3)
HSVの画像の形 : (1520, 2028, 3)
ある画素のHSV: [110  180  200]
```

形は同じ `(縦, 横, 3)` のままですが、**中身の3つ組の意味が変わった**ことが大事です。BGR では「青・緑・赤の明るさ」でしたが、HSV では「色相・彩度・明度」になっています。`[110, 180, 200]` なら「色相110（青あたり）・彩度180（そこそこ鮮やか）・明度200（明るめ）」という意味です。

## 2.5 H・S・V を1枚ずつ見てみる

「色味と明るさが分かれている」と言われても、まだピンと来ないと思います。**H・S・V をそれぞれ別の白黒画像として保存して、目で見比べて**みましょう。第1章で学んだスライス（`[:, :, 番号]`）を使います。

```python
# step6.py
import cv2

img = cv2.imread("mat.jpg")
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# 3つ組のそれぞれを取り出す
h = hsv[:, :, 0]   # 色相
s = hsv[:, :, 1]   # 彩度
v = hsv[:, :, 2]   # 明度

cv2.imwrite("hsv_h.jpg", h)
cv2.imwrite("hsv_s.jpg", s)
cv2.imwrite("hsv_v.jpg", v)
print("hsv_h.jpg / hsv_s.jpg / hsv_v.jpg を保存しました。")
```

3枚を VS Code で開いて見比べてください。読み取り方は次のとおりです。

- **hsv_v.jpg（明度）**：ふつうの白黒写真のように見えます。明るい場所が白く、影が黒くなります。「明るさの情報」がここに集まっている証拠です。
- **hsv_h.jpg（色相）**：明るさとは無関係に、色の種類で濃淡が決まります。マットの部分は、日なたでも日かげでも**同じくらいの濃さ**で写るはずです。これが「色味は明るさに左右されにくい」という HSV の長所です。
- **hsv_s.jpg（彩度）**：鮮やかな色付きマットは明るく、白っぽい・灰色っぽい部分は暗く写ります。

> **やってみよう 1**
> マットの上の「日なたの部分」と「影の部分」を1か所ずつ選び、`hsv` 画像でその位置の H と V を表示して比べてみましょう（`print(hsv[行, 列])`）。**V は大きく違うのに、H は近い値**になっていれば、HSV の狙いが体感できたことになります。

## 2.6 「色味で見分ける」を体験する

明るさを無視して色味だけで見分けられることを、もう少しはっきり体験してみましょう。同じ場所を、明るさを変えて2枚撮ってみます。

```bash
rpicam-jpeg -o mat_bright.jpg --timeout 2000
# （部屋を少し暗くする、または日かげにするなどして）
rpicam-jpeg -o mat_dark.jpg --timeout 2000
```

```python
# step7.py
import cv2

for name in ["mat_bright.jpg", "mat_dark.jpg"]:
    img = cv2.imread(name)
    hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
    # マットの中央あたりの画素を見る（座標は写真に合わせて調整）
    print(name, "の中央付近 HSV:", hsv[760, 1000])
```

2枚の出力を比べると、**V（3つ目）は明るさで大きく動くのに、H（1つ目）はあまり動かない**はずです。`leaf_monitor.py` がマットの色を学習するとき、H（色相）を主役に、S・V には広めの許容幅を持たせているのは、まさにこの性質を利用しているからです（詳しくは第3章で扱います）。

> **やってみよう 2**
> もし H が2枚で大きく違ってしまったら、それはホワイトバランス（カメラが自動で色味を調整する機能）が効きすぎている可能性があります。撮影のたびに色味がずれると計測も不安定になります。「撮影条件をなるべく一定にする」ことがなぜ大切か、考えてみましょう。

## 2.7 `leaf_monitor.py` への橋渡し

この章の `cv2.cvtColor(..., cv2.COLOR_BGR2HSV)` は、`leaf_monitor.py` の至るところで使われています。

**(1) 解析するとき** — `analyze()` の中

```python
hsv = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2HSV)
```

撮った写真をまず HSV に変換しています。私たちが `step5.py` でやったことと同じです。

**(2) マットの色を学習するとき** — `calibrate()` の中

```python
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
Hc, Sc, Vc = hsv[:, :, 0], hsv[:, :, 1], hsv[:, :, 2]
```

`step6.py` でやったのと同じく、H・S・V を3つに分けています。`leaf_monitor.py` はこのあと、H（色相）の最も多い値を「マットの色」と判断します。鉢が画面の真ん中にあっても、面積が一番大きいマットの色がちゃんと選ばれる、という賢いやり方です。

仕様書の「3.3 背景差分 → 色付きマット方式へ」「3.1 古典的画像処理を採用」で説明されている設計は、この HSV 変換があってこそ成り立っています。**明るさ（V）が多少変わっても色相（H）はほぼ一定**という、いまこの章で目で確かめた性質が、そのまま根拠になっているのです。

---

## 2.8 この章のまとめ

- BGR は「色味」と「明るさ」が混ざっているため、明るさの変わる屋外では色の指定が難しい。
- **HSV** は色を **H（色相）・S（彩度）・V（明度）** に分け、明るさは V だけが担当する。
- だから「明るさが変わっても、色味（H）で物を見分ける」ことができる。
- OpenCV の H は **0〜180**、S と V は 0〜255 という範囲に注意。
- `cv2.cvtColor(img, cv2.COLOR_BGR2HSV)` で変換し、`[:, :, 番号]` で H・S・V を別々に取り出して確認できる。

## 2.9 確認問題

1. 同じ青いマットなのに、日なたと日かげで BGR の `B` の値が大きく変わってしまう。HSV にすると、主にどの要素（H・S・V）がこの明るさの違いを引き受けるか答えなさい。
2. OpenCV における色相 H が取りうる値の範囲を答えなさい。また S・V の範囲も答えなさい。
3. `cv2.cvtColor(img, cv2.COLOR_BGR2HSV)` の戻り値の `shape` は、もとの `img.shape` と比べてどうなるか説明しなさい。
4. `leaf_monitor.py` がマットの色を「色相（H）の最も多い値」から学習しているのはなぜ良い方法か、この章で学んだ HSV の性質を使って説明しなさい。
